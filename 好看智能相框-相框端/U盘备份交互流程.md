# 电子相框 U 盘管理 — 完整交互流程

---

## 一、整体流程概览

```
插入U盘 → 弹出引导气泡 + 后台扫描照片 → 点击"去管理"进入设置页
  → U盘照片管理页（扫描中 / 空U盘 / 照片浏览）
  → 选择关联账号（已绑定用户列表 / 去绑定账号）
  → 同步照片到相框（全部同步 / 按组同步 / 多选同步 / 单张同步）
  → 后台执行同步 → 完成/部分失败 → 结果通知
```

> **核心概念变更：** 同步的目标是将 U 盘照片"同步到相框"（关联到已绑定家庭成员的账号），而非直接"备份到云端"。用户通过选择一个已绑定的家庭成员账号作为同步目标。

## 二、页面与功能结构

### 2.1 相框播放画面（主界面）

主界面为全屏照片轮播，叠加以下浮层元素：

| 元素 | 位置 | 说明 |
|------|------|------|
| 背景照片轮播 | 全屏 | 5 秒自动切换，淡入淡出过渡 |
| U 盘引导气泡 | 底部居中 | 插入 U 盘后显示，含"去管理"按钮 |
| 后台同步状态指示器 | 右上角 | 蓝色呼吸圆点 + "后台同步中..."，仅同步期间显示 |
| 同步完成通知 | 右上角 | 绿色/橙色圆点 + 结果文字 + 关闭按钮，15 秒后自动消失 |
| 设置入口按钮 | 右下角 | 齿轮图标，可叠加红色未读圆点 |
| 模拟操作台 | 底部居中 | 开发调试用，正式环境隐藏 |

### 2.2 设置页面结构

全屏覆盖式页面，左右分栏布局：

| 区域 | 说明 |
|------|------|
| 左侧导航栏 | 宽 320px，含返回按钮 + 导航菜单项 |
| 右侧内容区 | 根据当前选中标签页显示对应内容 |

**导航菜单项：**

| 菜单项 | 可见条件 | 说明 |
|--------|---------|------|
| U 盘照片管理 | 仅 U 盘插入后可见 | 管理 U 盘中的照片 |
| 其他设置 | 始终可见 | 系统其他设置项 |

### 2.3 U 盘照片管理页

该页面根据 U 盘扫描状态展示三种不同界面：

```
┌─────────────────────────────────────────────────────────┐
│  标题栏：U 盘照片管理            关联账号：[头像] 用户名 ▼ │
├─────────────────────────────────────────────────────────┤
│                                                         │
│  状态 A：扫描中 UI（进度条 + 状态文字）                    │
│  状态 B：空 U 盘 UI（空状态插图 + 提示文字）               │
│  状态 C：照片浏览器（工具栏 + 照片网格）                   │
│                                                         │
└─────────────────────────────────────────────────────────┘
```

## 三、详细交互步骤

### 1. 插入 U 盘

| 项目 | 说明 |
|------|------|
| **触发条件** | 系统检测到 U 盘物理插入 |
| **后台行为** | 立即在后台开始扫描 U 盘中的所有照片文件 |
| **前台表现** | ① Toast 提示"检测到新 U 盘连接"（3 秒） ② 底部弹出引导气泡 |
| **侧边栏变化** | 设置页左侧导航栏中出现"U 盘照片管理"入口（仅 U 盘插入后可见） |

### 2. U 盘引导气泡

插入 U 盘后在相框播放画面底部居中弹出，带弹性入场动画。

```
┌──────────────────────────────────────────────────┐
│  [U盘图标]   发现 U 盘              [ 去管理 ]    │
└──────────────────────────────────────────────────┘
```

| 元素 | 说明 |
|------|------|
| U 盘图标 | 白色圆形背景 + U 盘 SVG 图标 |
| 标题 | **"发现 U 盘"**，白色粗体大字 |
| 操作按钮 | **"去管理"** — 白底蓝字圆角按钮，点击进入 U 盘管理 |
| 样式 | 蓝色渐变背景（`bg-blue-600`），大圆角，带阴影和白色半透明边框 |

> **交互说明：**
> - 引导气泡常驻显示，不自动消失，直到用户点击"去管理"或手动拔出 U 盘
> - 不影响照片轮播正常播放
> - 点击"去管理"后气泡关闭，自动打开设置页并切换到 U 盘照片管理标签页

### 3. 进入 U 盘管理

用户可通过以下方式进入 U 盘管理页：

| 入口 | 说明 |
|------|------|
| 引导气泡"去管理"按钮 | 关闭气泡 → 打开设置页 → 自动切换到 U 盘照片管理 |
| 设置页左侧"U 盘照片管理"导航项 | 仅 U 盘插入后可见，点击切换到对应标签页 |
| 设置入口按钮（右下角齿轮） | 打开设置页，如有 U 盘插入则自动切换到 U 盘管理；如有同步结果则自动展示结果提示并清除红点 |

### 4. 关联账号选择器

位于 U 盘照片管理页标题栏右侧，始终显示。

#### 4.1 未选择关联账号

```
└─────────────────────────────────────┘
```

| 元素 | 说明 |
|------|------|
| 标签 | "关联账号：" 灰色文字 |
| 状态文字 | **"请选择"** 蓝色粗体，提示用户操作 |
| 下拉箭头 | 灰色向下箭头 |
| 点击行为 | 弹出用户选择弹窗 |

#### 8.2 工具栏

**普通模式：**

| 左侧 | 右侧 |
|------|------|
| "共 **N** 张照片" | **"全部同步到相框"** 蓝色按钮 + **"多选"** 边框按钮 |

**多选模式：**

| 左侧 | 右侧 |
|------|------|
| **"全选"** 蓝色链接 + "已选择 **N** 张" | **"取消选择"** 红色边框按钮（"多选"按钮替换） |

> 多选模式下"全部同步到相框"按钮隐藏，同步和删除操作移至底部操作栏。

#### 8.3 照片分组

照片按**月份**分组显示（默认按时间**倒序**排列，最新的在最上面），每个分组包含：

| 元素 | 说明 |
|------|------|
| 分组标题 | 如"2024年12月"，粗体 |
| 照片计数 | 如"12 张"，灰色小字 |
| 分组操作按钮 | 根据同步状态变化（见下方状态表） |
| 照片网格 | 自适应列数，最小宽度 170px，正方形缩略图，圆角 16px |

**分组操作按钮状态：**

| 状态 | 按钮样式 | 说明 |
|------|---------|------|
| 默认 | **"同步到相框"** 蓝色按钮 | 点击同步该组所有照片 |
| 正在同步本组 | **"同步中..."** 灰色 + 旋转图标 | 不可点击 |

> **注意：** 由于同步成功的照片会自动从列表中移除，当一个分组的所有照片都同步成功后，该分组会自动消失（无需"已全部同步"状态）。当所有分组消失（即所有照片都已同步），自动切换到"所有照片已同步完成"的空状态页面。

> 多选模式下分组操作按钮隐藏，替换为分组全选复选框。

#### 8.4 照片缩略图

每张照片缩略图包含以下元素：

| 元素 | 位置 | 说明 |
|------|------|------|
| 缩略图 | 填满 | 正方形裁切，`object-fit: cover` |
| 选择复选框 | 左上角 | 仅多选模式显示，圆形，选中时蓝色填充 + 白色对勾 |

> **注意：** 列表中只展示未同步的照片，同步成功的照片会自动从列表中移除，因此不需要同步状态图标。同步过程中正在处理的照片会显示为半透明。

**交互行为：**

| 模式 | 点击行为 |
|------|---------|
| 普通模式 | 打开全屏照片查看器 |
| 多选模式 | 切换选中/取消选中状态 |

#### 8.5 多选模式底部操作栏

进入多选模式后在页面底部显示固定操作栏。

```
┌───────────────────────────────────────────────────────┐
│  已选择 N 张              [同步到相框]  [删除选中]      │
└───────────────────────────────────────────────────────┘
```

| 元素 | 说明 |
|------|------|
| 选中计数 | "已选择 **N** 张"，蓝色数字 |
| 同步到相框按钮 | 蓝色，含云朵图标，选中数为 0 时置灰不可点击 |
| 删除选中按钮 | 红色，含垃圾桶图标，选中数为 0 时置灰不可点击 |

**多选操作说明：**

| 操作 | 说明 |
|------|------|
| 点击照片 | 切换选中/取消选中 |
| 分组复选框 | 全选/取消全选该分组内的所有照片 |

## 四、同步工作执行逻辑（技术细节）

同步的本质是：**使用已绑定家庭成员的账号凭证，将 U 盘中的照片上传到该账号的云端相册，并同步到相框显示**。完全复用手机端的批量发布流程（`UploadImageTask` → `OssManager` → `UploadImgApi`），Pad 端仅替换凭证来源。

### 账号与凭证管理

| 项目 | 说明 |
|------|------|
| **账号来源** | 通过手机 APP 扫码绑定到相框的家庭成员账号 |
| **存储方式** | 已绑定用户列表存储在本地，包含 `userId`、`name`、脱敏手机号、头像 URL |
| **关联选择** | 用户在 U 盘管理页面选择一个已绑定账号作为同步目标 |
| **凭证获取** | 使用选中用户的 `userId` + `token` 进行上传和发布 |
| **生命周期** | 绑定关系长期保留，直到用户在 APP 端解绑 |

### 整体执行流程

```
1. 获取 STS 凭证（refreshStsCredentialsBeforePublish）
2. 提取照片 EXIF 元信息 + 计算文件 MD5
3. 将照片列表按每 10 张分批（BATCH_SIZE = 10）
4. 对每一批执行：上传 OSS → 调用发布接口
5. 全部批次完成 → 更新状态 → 展示结果
```

### 步骤 1：获取 STS 凭证

| 项目 | 说明 |
|------|------|
| **接口** | `POST /rlm/social/authFileToken` |
| **参数** | `userId`（关联账号）、`token`（关联账号） |
| **返回** | `AccessKeyId`、`AccessKeySecret`、`SecurityToken`、`Expiration`（秒级时间戳） |
| **有效性判断** | `Expiration > 当前时间 + 300秒`（5 分钟缓冲），过期则重新获取 |
| **存储** | 存储在内存中（`@Volatile` 变量），发布前刷新、上传失败时刷新 |

### 步骤 2：提取照片元信息

对应手机端 `ImageMetadataExtractor.extractImageMetadata()`，从照片 EXIF 中提取以下信息：

| EXIF 标签 | 提取字段 | 兜底策略 |
|-----------|---------|---------|
| `TAG_IMAGE_WIDTH` / `TAG_IMAGE_LENGTH` | `width`、`height` | 通过 BitmapFactory 解码获取 |
| `TAG_ORIENTATION` | `degree`（0/90/180/270） | 用于宽高交换判断 |
| `getLatLong()` | `shootXY`（"纬度,经度"） | 无则留空 |
| `TAG_DATETIME` | `dateTime`（yyyy-MM-dd HH:mm:ss） | 无 EXIF 时使用文件修改时间 |
| `TAG_MAKE` + `TAG_MODEL` | `cameraInfo` | 无则留空 |
| `TAG_FOCAL_LENGTH` | `focalLength`（如 "4.25mm"） | 无则留空 |
| `TAG_EXPOSURE_TIME` | `shutter`（如 "1/125s"） | 无则留空 |
| `TAG_F_NUMBER` | `aperture`（如 "f/1.8"） | 无则留空 |
| `TAG_ISO_SPEED_RATINGS` | `iso`（如 "ISO 200"） | 无则留空 |

同时计算文件 MD5，用于生成 OSS objectKey 和去重：
- **objectKey 格式**：`release/image/{文件MD5}.{扩展名}`（如 `release/image/abc123def.jpg`）
- 相同文件内容 → 相同 objectKey → OSS 自动去重

### 步骤 3：批量上传 OSS

对应手机端 `AliOssUpload` 的逻辑：

| 项目 | 说明 |
|------|------|
| **上传方式** | 阿里云 OSS `MultipartUploadRequest`（分片上传） |
| **分片大小** | 100KB（`100 * 1024`） |
| **最大并发请求** | 5（`maxConcurrentRequest = 5`） |
| **SDK 自动重试** | 3 次（`maxErrorRetry = 3`） |
| **连接/Socket超时** | 均为 15 秒 |
| **Bucket** | `hksocial`（公开）/ `hksocial-private`（私有） |
| **Endpoint** | `https://oss-accelerate.aliyuncs.com` |
| **OSS 回调** | 上传完成后 OSS 自动回调 `{urlHost}/social/upload/advance`，body 为 `{"EventType":"image","FileName":"{objectKey}"}` |

**去重优化**：上传前先调用 `doesObjectExist(bucket, objectKey)` 检查 OSS 上是否已有同名文件，已存在则直接跳过上传，标记为成功。

**STS 过期处理**：上传过程中遇到 403 / `InvalidAccessKeyId` 错误，自动调用 `StsApi.getAliYunSTS()` 刷新凭证后重试，最多重试 5 次。

### 步骤 4：调用发布接口

对应手机端 `BaseUploadTask.releaseBatchImages()` + `UploadImgApi.releaseImgs()`：

| 项目 | 说明 |
|------|------|
| **接口** | `POST /rlm/release_batch/v1` |
| **批次大小** | 每 10 张照片调用一次（`BATCH_SIZE = 10`） |
| **batchId** | `System.currentTimeMillis()`，同一次同步任务的所有批次共用 |
| **displayTotal** | 本次同步的照片总数（所有批次共用） |
| **albumIds** | 目标相册 ID |

**请求参数**：

| 参数 | 来源 | 说明 |
|------|------|------|
| `userId` | 关联账号 | 用户 ID |
| `token` | 关联账号 | 用户 Token |
| `albumIds` | 目标相册 | 逗号分隔的相册 ID |
| `batchId` | 同步开始时间戳 | 标识同一次同步任务 |
| `displayTotal` | 照片总数 | 所有批次共用 |
| `imageList` | `List<ReleaseImgChild>` | 本批次的照片元数据列表 |

**`ReleaseImgChild` 字段**：

| 字段 | 说明 |
|------|------|
| `workType` | 固定 `"2"` |
| `flag` | 固定 `"2"` |
| `coverFileName` | OSS objectKey（`release/image/{md5}.{ext}`） |
| `coverW` / `coverH` | 照片宽高 |
| `shootTime` | 拍摄时间 |
| `shootaddr` / `addr` | 拍摄地址 |
| `shootXY` / `lanlon` | GPS 坐标（"纬度,经度"） |
| `cameraInfo` | 相机型号 |
| `focalLength` / `aperture` / `shutter` / `iso` | 拍摄参数 |
| `poiTitle` | POI 名称 |
| `province` / `city` / `county` | 行政区划 |
| `country` | 固定 `""` |
| `userId` | 关联账号的 userId |
| `imageClientId` | 文件 MD5（服务端用于去重） |

### 步骤 5：失败重试机制

| 场景 | 策略 |
|------|------|
| **OSS 上传失败** | SDK 层自动重试 3 次（`maxErrorRetry = 3`） |
| **STS 凭证过期（403）** | 自动刷新 STS 后重试，最多 5 次 |
| **发布接口失败** | 批次级别重试，最多 3 次（`MAX_RETRY_COUNT = 3`），间隔 2 秒 |
| **发布重试条件** | 超时、`SocketException`、`UnknownHostException`、`ConnectException` |
| **Token 过期** | 暂停上传，提示用户重新绑定或刷新凭证 |
| **全部批次失败** | 提示"图片发布失败（所有批次都失败）" |

### 进度计算

| 阶段 | 占比 |
|------|------|
| OSS 上传 | 92%（`UPLOAD_PROGRESS_RATIO = 0.92f`） |
| 接口发布 | 5%（`RELEASE_PROGRESS_RATIO = 0.05f`） |
| 收尾处理 | 3%（至 `MAX_PROGRESS_BEFORE_COMPLETE = 0.97f`，完成时跳到 100%） |

> **完全复用手机端批量发布逻辑**（`UploadImageTask` → `AliOssUpload` → `UploadImgApi`），接口调用方式和参数格式一致，唯一区别是 `userId` + `token` 来源于已绑定的家庭成员账号凭证。

## 五、边界场景与异常处理

### U 盘相关

| 场景 | 处理方式 |
|------|----------|
| **同步过程中拔出 U 盘** | 立即暂停同步任务，弹出提示"U 盘已断开，同步已暂停"。已上传到 OSS 的照片保留，未上传的标记为中断。设置侧边栏隐藏"U 盘照片管理"入口 |
| **拔出后重新插入同一 U 盘** | 重新扫描，通过文件哈希识别已同步过的照片，仅显示新增/未完成的照片数量 |
| **插入不同 U 盘** | 视为全新 U 盘，重新扫描全部照片 |
| **U 盘中无照片** | 扫描完成后显示空 U 盘 UI："未检测到照片"，提示支持的格式 |
| **U 盘文件损坏** | 跳过无法读取的文件，扫描完成后仅展示可读取的照片数量 |

### 账号与绑定相关

| 场景 | 处理方式 |
|------|----------|
| **无已绑定用户 + 触发同步** | 弹出用户选择弹窗（空状态），引导去绑定账号 |
| **绑定成功后有待同步照片** | 自动关联新绑定用户，继续之前的同步流程 |
| **绑定过程中取消** | 关闭弹窗，待同步照片列表清空 |
| **同步中尝试切换关联账号** | 选择器半透明不可点击，Toast 提示"同步进行中，请稍后再操作" |
| **切换关联账号** | 弹出用户选择弹窗（标题"切换关联账号"），选择后更新显示，Toast 确认 |
| **同步完成后列表为空** | 自动切换到"所有照片已同步完成"空状态页面 |

### 照片管理相关

| 场景 | 处理方式 |
|------|----------|
| **删除所有照片后** | 自动退出多选模式，切换到空 U 盘 UI |
| **全屏查看时删除当前照片** | 如还有剩余照片，切换到第一张；如无剩余，关闭查看器 |
| **照片数为 0 时点同步** | Toast 提示"暂无照片可同步" |

### 页面导航相关

| 场景 | 处理方式 |
|------|----------|
| **同步中关闭设置页** | 同步在后台继续执行，右上角显示"后台同步中..."指示器，Toast 提示"同步在后台继续，完成后会通知你" |
| **同步中切换到其他设置标签** | 同步不受影响，切回 U 盘管理时恢复显示当前进度 |
| **同步完成时用户不在设置页** | 相框播放画面右上角显示完成通知，15 秒后消失并在设置入口显示红点 |
| **关闭设置页时处于多选模式** | 自动退出多选模式，清空选中状态 |

## 六、通知类型汇总

| 通知类型 | 触发时机 | 展示方式 | 持续时间 |
|---------|---------|---------|---------|
| 检测到 U 盘 | 插入 U 盘 | Toast 提示"检测到新 U 盘连接" | 3 秒 |
| 引导气泡 | 插入 U 盘 | 底部浮动气泡"发现 U 盘" + "去管理" | 常驻（直到用户操作或拔出） |
| 关联成功 | 选择/切换关联账号 | Toast 提示"已切换关联账号：XXX" | 3 秒 |
| 绑定成功 | 扫码绑定完成 | Toast 提示"绑定成功！已关联账号：XXX" | 3 秒 |
| 同步进行中锁定提示 | 同步中尝试切换账号 | Toast 提示"同步进行中，请稍后再操作" | 3 秒 |
| 后台同步中 | 同步执行期间 | 右上角蓝色呼吸圆点 + "后台同步中..." | 跟随同步状态 |
| 暂停 | 用户点击暂停 | Toast 提示"已暂停同步" | 3 秒 |
| 恢复 | 用户点击继续 | Toast 提示"已恢复同步" | 3 秒 |
| 同步已取消 | 用户取消同步 | Toast 提示"同步任务已取消" | 3 秒 |
| **同步完成（全部成功）** | 同步全部完成 | 右上角绿色通知"同步完成 · N 张照片已备份" + 设置页内绿色结果条 | **15 秒自动消失** → 设置红点 |
| **同步完成（部分失败）** | 同步完成但有失败 | 右上角橙色通知"同步部分完成 · 成功 X 张，失败 Y 张" + 设置页内橙色结果条（含重试按钮） | **15 秒自动消失** → 设置红点 |
| 暂无照片可同步 | 列表为空时触发同步 | Toast 提示"暂无照片可同步" | 3 秒 |
| 删除成功 | 确认删除照片 | Toast 提示删除结果（区分是否删除原始文件） | 3 秒 |
| 网络异常 | 网络断开 | 右上角黄色指示器 | 跟随网络状态 |
| U 盘断开 | 同步中拔出 U 盘 | Toast 提示 + 暂停同步 | 3 秒（提示） |

## 八、支持的照片格式

| 格式 | 支持条件 |
|------|---------|
| JPG / JPEG | 所有设备 |
| PNG | 所有设备 |
| BMP | 所有设备 |
| HEIC / HEIF | Android 9（API 28）及以上，或非 Android 设备 |

> **扫描逻辑：**
> - 扫描应递归遍历 U 盘中所有文件夹
> - 去重机制：计算文件 MD5 生成 objectKey（`release/image/{MD5}.{ext}`），上传前通过 OSS `doesObjectExist()` 检查是否已存在，已存在则跳过。同时 `imageClientId`（文件 MD5）发送给服务端做二次去重

## 九、备注

- 整个同步发布流程完全复用手机端的批量发布逻辑，接口和参数格式一致，区别仅在于凭证来源（已绑定家庭成员账号）
- U 盘管理入口仅在 U 盘插入后出现在设置侧边栏，拔出后隐藏
- 同步任务全程后台执行，不阻塞相框正常播放照片
- 同步完成通知在主画面显示 15 秒后自动消失，通过设置入口红点引导用户查看详情
- 账号体系基于"绑定家庭成员"模式，通过手机 APP 扫码绑定相框，支持多个家庭成员
- 同步目标为选中的关联账号，同步过程中不可切换账号
- 调用上传相关接口时，`clientType` 仍使用 Pad 端的值（100），但 `userId` 和 `token` 使用关联账号的值
- 照片支持按月份分组浏览，按时间倒序排列（最新在前）
- 多选模式支持全选、分组全选、单张选择，选中照片可批量同步或删除
- 删除操作可选择是否同时删除 U 盘原始文件
- 同步成功的照片自动从列表中移除，列表仅展示未同步和同步失败的照片
- 所有照片同步完成后，自动切换到"所有照片已同步完成"空状态页面，提示用户可安全拔出 U 盘
