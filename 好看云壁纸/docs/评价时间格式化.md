# 评价时间展示规范（相对时间）

用于订阅详情页、评价列表等场景的**中文相对时间**文案，与客户端本地时区一致。

## 分档规则

| 条件 | 展示 |
|------|------|
| 距现在 &lt; 1 分钟 | **刚刚** |
| 1 分钟 ≤ 差值 &lt; 60 分钟 | **N 分钟前** |
| 今天内、≥ 1 小时且仍属「今天」自然日 | **N 小时前**（如 **3 小时前**） |
| 评论日期 = 本地日历的「昨天」 | **昨天**（可选细化：**昨天 HH:mm**） |
| 早于昨天、且在最近 7 个自然日内 | **N 天前**（如 **4 天前**） |
| ≥ 7 天且同年 | **M 月 D 日** |
| 非本年 | **YYYY 年 M 月 D 日** |

## 「昨天」与「N 小时前」

- **昨天**按**本地 0 点切日**，不按「满 24 小时」计算。
- **优先级（推荐）**：若评论落在「昨天」这一自然日，统一显示 **昨天**，不再显示「15 小时前」等，避免与「昨天」混用造成困惑。
- 若产品更强调时效，可改为：**未满 24 小时优先 N 小时前**；满 24 小时且为昨天则 **昨天**——选定一种后全站统一。

## 实现注意

- 基准：**当前设备本地时间**；服务端若给 UTC，需先换算再比日历日。
- **未来时间**（时钟误差）：可降级为 **刚刚** 或 **刚刚发布**。
- 分钟、小时、天数均用**正整数**展示（如 `3 小时前`，不写小数）。

## 示例

- 30 秒前 → **刚刚**
- 45 分钟前 → **45 分钟前**
- 今日 10:00 发布，当前今日 14:00 → **4 小时前**
- 本地「昨天」任意时刻 → **昨天**
- 5 天前的自然日 → **5 天前**
