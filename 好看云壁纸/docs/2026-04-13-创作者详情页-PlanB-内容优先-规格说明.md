# 好看云壁纸 — 创作者详情页（Plan B：内容浏览优先，点作品进合辑）

> 日期：2026-04-13  
> 目的：为“以创作者为核心”的展示逻辑提供一套可落地、可与 `creator-detail.html`（Plan A）对比讨论的方案，并用于生成 H5 原型文件。

---

## 1. 背景与目标

当前 `好看云壁纸/creator-detail.html`（Plan A）以「创作者 → 合辑列表 → 合辑详情 → 订阅」为主路径，更偏“订阅/合辑管理”导向。

Plan B 目标：

- 以“内容浏览”为第一驱动，让用户先看见作品、产生兴趣，再自然进入合辑完成订阅。
- 保留合辑入口，但降级为“次要模块”，避免与主内容流争夺注意力。
- 页面层级更接近原生 Push 栈，减少多层模态叠加带来的手势冲突与认知负担。

---

## 2. 信息架构与页面清单

### 2.1 页面 1：创作者详情页（内容优先）

- **顶部 sticky Header**
  - 创作者头像、昵称、短简介
  - 统计信息（建议 2~3 项）：合辑数 / 订阅数 / 作品数
  - 可选按钮：分享/搜索（原型可仅占位）

- **次要模块：合辑入口（横向）**（用户已确认“是”）
  - 横向滚动的小卡/胶囊，展示合辑封面缩略+合辑名（不放订阅按钮）
  - 点击进入页面 2（合辑详情页）

- **主区域：作品流（3 列网格）**
  - 展示该创作者所有合辑的作品缩略图（扁平化）
  - 每个作品角标（轻量）：
    - 左下：合辑名（截断）
    - 右上：更新频率/“新”（可选）
  - **点击作品**：直接进入页面 2（合辑详情页），并携带焦点作品索引（用于定位/高亮）

### 2.2 页面 2：合辑详情页（Push 右滑进入）

遵循 `好看云壁纸/index.html` 的二级页面规范（`translateX(100%) → 0`）。

- 合辑封面、合辑名、更新频率、张数
- 创作者信息区（可“回到创作者页”）
- 壁纸网格（3 列）
- 底部订阅栏：订阅 / 已订阅（可取消）
- 点击单张壁纸：弹出 action sheet（单独设置/加入轮播池），复用现有视觉与反馈

---

## 3. 关键交互与路由参数

### 3.1 入口（从内容进入创作者）

从 `index.html` 的内容卡片中，点击创作者信息区进入：

- 形式：`creator-detail-plan-b.html?creatorId=<id>&from=content`

### 3.2 点作品进入合辑

点作品跳合辑详情时，传递：

- `albumId=<id>`
- `focusIndex=<idxInAlbum>`

原型实现可用内存状态即可；如做 URL 也可以：

- `...#album/<albumId>?focus=<idx>`

---

## 4. 数据结构（原型内）

沿用 Plan A 的思路，用 mock 数据驱动：

- `CREATORS[creatorId]`
  - `id, name, avatar, cover, bio, desc, stats`
  - `albums[]`
- `album`
  - `id, name, count, update, updateColor, cover, wallpapers[]`

### 4.1 作品流聚合

由 `albums[].wallpapers[]` 扁平化得到 `creatorWorks[]`：

- `src`
- `albumId`
- `albumName`
- `idxInAlbum`

点击作品时：

- `openAlbumDetail(albumId, idxInAlbum)`

合辑详情页打开后：

- 滚动/定位到 `idxInAlbum` 对应壁纸，并做一次性高亮（可选）

---

## 5. 订阅规则（保持一致）

- 订阅粒度：**合辑**
- 状态：未订阅 / 已订阅（支持取消）
- 订阅成功反馈：Toast +（可选）触发锁屏引导弹窗（与 `index.html` 订阅链路一致）

---

## 6. 与 Plan A 的对比结论（用于讨论）

- **Plan A**：合辑优先，适合强订阅/强管理；但层级更深、模态叠加更重。
- **Plan B**：内容优先，适合“先种草再订阅”；对“从内容进入创作者”的叙事更自然，且二级页更贴近原生 Push。
