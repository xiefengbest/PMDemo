## 一、改动背景

Pad 端推荐方案从 V1（规则轮播）升级为 V2（AI 大模型智能推荐）。手机端需配合增加：
1. 首次启动隐私弹窗中**合并智能推荐说明**，用户同意后所有相框**默认开启**
2. 相框设置菜单中提供**关闭/开启开关**（关闭时自动清除推荐数据）

---

## 二、改动总览

| 改动区域 | 类型 | 说明 |
|---------|------|------|
| 首次启动隐私弹窗 | **修改** | 合并智能推荐隐私说明，同意后所有相框默认开启 AI 推荐 |
| 相框设置菜单 | **新增 Toggle** | 主设置卡片顶部，含"智能推荐"Toggle 开关，关闭时自动清除数据 |
| 设备数据模型 | **新增字段** | `aiRecommend`、`aiConsented` |

> **设计原则**：授权合并到首次启动隐私弹窗中一次性完成，不单独弹窗打扰用户；开关放在设置菜单中，不侵入内容浏览流。

---

## 三、详细交互说明

### 3.1 首次启动隐私弹窗（合并智能推荐说明）

在现有的"服务协议与隐私政策"弹窗底部，新增**智能推荐说明卡片**：

```
┌──────────────────────────────────┐
│  服务协议与隐私政策                 │
│                                  │
│  感谢您信任并使用好看智能相框...     │
│  《用户协议》《隐私政策》           │
│                                  │
│  ┌──────────────────────────┐   │
│  │ ✦ 每日精选推荐            │   │  ← 淡蓝色背景卡片
│  │ 相框会从已有照片中，根据   │   │
│  │ 季节、节日等场景每天自动   │   │
│  │ 挑选最合适的进行展示。仅   │   │
│  │ 分析标签信息，不会读取照   │   │
│  │ 片内容。随时可在设置关闭。 │   │
│  └──────────────────────────┘   │
│                                  │
│  ┌────── 同意并继续 ──────┐      │
│  └────── 不同意 ──────────┘      │
└──────────────────────────────────┘
```

**卡片样式**：
- 背景色 `#F5F7FF`，圆角 `12px`，1px 边框 `rgba(99,102,241,0.1)`
- 顶部：Indigo 色星芒图标 + "每日精选推荐"标题
- 正文：以用户视角描述功能价值，明确"从已有照片中挑选"，强调"仅分析标签信息，不读取照片内容"，告知可随时关闭

**文案原则**：
- 先讲功能好处（自动挑选照片），而非技术实现
- 用"匿名化的时间和场景信息"代替"脱敏元数据发送至 AI"等技术术语
- 用正面表述（"照片始终留在本地"）代替负面清单（不传原图、不传位置...）
- 给用户掌控感（"随时关闭"）

**交互逻辑**：
- 用户点击"同意并继续" → 所有已绑定及后续绑定的相框自动 `aiConsented=true, aiRecommend=true`
- 用户后续可在单台相框的设置中关闭

### 3.2 智能推荐开关（相框设置菜单）

**位置**：设置菜单 ActionSheet 主卡片内，**第一行**

**结构**：
```
┌─────────────────────────────────────┐
│            相框设置                    │
│─────────────────────────────────────│
│      ✦ 智能推荐  [ ● ]              │  ← 与下方选项同风格
│─────────────────────────────────────│
│      ✎ 重命名相框                     │
│      ⌖ 修改位置                      │
│      ↻ 重置人物相册                   │
│─────────────────────────────────────│
│      ⏻ 解绑相框（红色）               │
└─────────────────────────────────────┘
│            取消                       │
└─────────────────────────────────────┘
```

**样式规范**（与其他选项完全统一）：
- 居中排列：图标 + "智能推荐"文字 + Toggle 开关，使用 `flex justify-center gap-2`
- 文字颜色：`text-blue-600`，与其他操作项一致
- 图标：AI 星芒图标，使用 `currentColor`（蓝色），不单独设色
- Toggle 尺寸：`42×25`（比之前的 `51×31` 更紧凑，融入行内不突兀）
- 内边距：`py-3.5`，与其他项一致
- 无额外状态标签（Toggle 本身已表达开/关状态）

**开关状态**：

| 状态 | Toggle 背景色 |
|-----|-------------|
| 关闭 | 灰色 `#E5E7EB` |
| 开启 | 蓝色 `#3B82F6` |

**交互逻辑**：
- 开 → 关：关闭智能推荐并**自动清除推荐数据**（`aiRecommend=false, aiConsented=false`），Toast "已关闭智能推荐，推荐数据已清除"
- 关 → 开：直接开启（隐私协议已在首次启动时同意），Toast "已开启智能推荐"

> **设计说明**：智能推荐行遵循 ActionSheet 统一视觉规范——居中布局、蓝色文字、同等间距，仅比普通选项多一个内联 Toggle 组件作为状态指示器，不破坏整体一致性。

## 四、数据模型变更

### FrameDevice 新增字段

| 字段 | 类型 | 默认值 | 说明 |
|-----|------|-------|------|
| `aiRecommend` | boolean | `false` | 智能推荐是否开启 |
| `aiConsented` | boolean | `false` | 用户是否已完成隐私授权 |

### API 接口（建议）

| 接口 | 方法 | 说明 |
|-----|------|------|
| `/api/device/{id}/ai-recommend` | PUT | 开启/关闭智能推荐，body: `{ enabled: boolean }`。关闭时后端自动清除推荐缓存和授权记录 |
| `/api/device/{id}/ai-recommend/consent` | POST | 记录用户授权 |

## 八、Android / iOS 实现注意事项

### 通用
- Toggle 开关使用系统原生 Switch 组件
- `aiConsented` 状态建议持久化到服务端（非本地），确保换手机后状态同步
- 关闭开关时需调用后端接口清除推荐缓存和授权记录

### Android
- Toggle 使用 `SwitchCompat`，自定义 track 和 thumb 颜色

### iOS
- Toggle 使用原生 `UISwitch`，设置 `onTintColor = .systemBlue`
