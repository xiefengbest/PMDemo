## 1. 换账号迁移（前后清单）

**背景**：Cursor **没有**官方的「记忆 / User Rules 一键迁到新账号」。换号后云端 **Memories**、旧号下的 **User Rules** 一般**不会**自动跟随；对话历史通常也**无法**迁移。

### 换账号之前（旧账号仍能登录）

| 步骤 | 做什么 |
|------|--------|
| ① User Rules | 设置里打开 **Rules / User Rules**，**全选复制** → 粘贴进 **`USER_RULES.md`**（或另存 `USER_RULES_FROM_CURSOR.txt`）。 |
| ② Memories | 设置里找到 **Memories**，**逐条复制** → 粘贴到本文下方 [§3 我的记忆条目](#3-我的记忆条目)。无列表时从旧对话里摘抄「已记住」的要点。 |
| ③ 其他偏好 | 填写本文 [§4](#4-其他偏好模型--mcp--账号)（**不要写密钥**）。 |
| ④（可选）本机库 | macOS：`~/Library/Application Support/Cursor/User/globalStorage/state.vscdb` 可复制一份到桌面备查；**勿**用其覆盖新账号同名文件。 |
| ⑤ 仓库文件 | 确认已保存：`用户上下文汇总.md`、`USER_RULES.md`、`.cursor/rules/user-context-from-repo.mdc`。 |

### 换账号之后（新账号）

1. 登录新号，确认订阅/额度（含高级模型）。
2. 将 **`USER_RULES.md`** 全文粘贴到新账号 **User Rules**（跨项目一致时推荐）。
3. 打开本仓库：项目规则会加载，助手会参考本汇总与 `USER_RULES.md`。
4. 把 [§3](#3-我的记忆条目) 中的条目在新账号 **Memories** 里**重新添加**（若支持）。
5. **MCP / API Key** 在新号里重新配置；密钥勿提交 Git。

### 换号后仍可能丢的

- 旧账号**云端聊天记录**、旧团队**团队规则**、未写入本文或 `USER_RULES.md` 的条目。

## 2. 记忆：如何备份与填写

- 记忆绑定**旧账号云端**，助手**不能**自动导出；必须在**仍登录旧号**时手动复制。
- 建议每条带 **日期** 或标签（如 `#工作流` `#技术栈`）。
- **禁止**写入密钥、Token、未公开内网地址；可写「密钥放在环境变量 XXX」。
