# Pad 端每日智能推荐 V2 方案更新同步

**文档版本**：V2（2026-03-16 更新）
**涉及产品**：好看智能相框 - 相框端
**完整方案文档**：`pad_daily_recommendation_v2.md`

---

## 一、方案概述

V2 将相框的每日照片推荐从"规则轮播"升级为 **AI 大模型智能推荐**。核心变化是：将照片元数据、今日上下文、用户偏好组装后喂给 AI（如 DeepSeek），由 AI 直接输出每日推荐的 100 张照片及展示顺序。开发者不写推荐算法，只维护数据组装和 Prompt。

**一句话总结**：让相框像一位"懂你的家人"来挑选今天要看的照片，同时确保数据全链路隐私合规。

---

## 二、核心能力变化

| 维度 | V1 | V2 |
|-----|----|----|
| 选片逻辑 | 按时间顺序轮播 | AI 综合分析，智能推荐 |
| 内容理解 | 无 | AI 理解场景、人像、大区等属性 |
| 上下文感知 | 仅区分新旧照片 | 感知季节、节日、**相框地点**、天气 |
| 用户偏好 | 不参考 | 参考点赞、展示历史等行为信号 |
| 隐私合规 | 无特殊处理 | **数据脱敏 + 用户授权 + 服务商 DPA** |
| 策略迭代 | 改代码重新部署 | 改 Prompt 即可生效 |
| Pad 端改动 | — | **零改动**，API 完全兼容 |

## 三、本次重点更新（两大补充）

### 3.1 新增：相框地点感知

V2 方案新增了**相框所在地点**作为推荐上下文，具体影响：

- **今日上下文**新增 `frameCity`（相框城市）、`frameProvince`（省份）、`weather`（天气）三个字段
- AI Prompt 新增两条推荐规则：**季节与天气关联**（雨天推温馨室内照）、**地点亲和**（相框在杭州时优先推杭州拍的照片）
- 季节判断按相框所在地气候区修正（如哈尔滨 3 月仍为冬季），节日检测覆盖区域性节日

**需要配合**：设备服务需提供相框绑定城市信息的查询接口。

### 3.2 新增：隐私合规与数据安全体系

由于方案涉及将用户数据传给第三方 AI 服务，我们设计了完整的隐私合规方案：

**数据脱敏策略**（传给 AI 前在服务端完成）：

| 原始数据 | 脱敏处理 | 传给 AI 的值 |
|---------|---------|------------|
| 照片真实 ID | 替换为临时索引 | `idx: 1, 2, 3...` |
| 人脸检测（hasFace） | 模糊化 | `hasPortrait: true/false` |
| 精确城市（杭州） | 模糊化为大区 | `region: 华东` |
| 家庭成员名（妈妈） | 代号替换 | `author: A` |

核心原则：**传给第三方 AI 的数据不可逆向识别具体个人**。

**合规要求**：

- 需与 AI 服务商签订**数据处理协议（DPA）**，禁止数据用于模型训练、调用后即销毁
- 优先选用**国内模型**（DeepSeek / Qwen / GLM），避免数据出境风险
- App 端需新增 AI 推荐**用户授权弹窗**（首次启用时弹出，可随时关闭）
- 需更新 App **隐私政策文档**
- 上线前需完成**个人信息保护影响评估（PIA）**

## 四、各团队关注事项

### 服务端团队

- 新增 5 个类、修改 2 个类（详见方案 12.2 节）
- 核心工作：数据脱敏组装 → 调 AI API → 缓存结果 → idx 映射还原
- 支持灰度发布（按 userId 分流）和降级回退 V1
- 每用户每日仅调用 1 次 AI，后续请求走 Redis 缓存

### App 端团队（手机端）

- 需新增 **AI 智能推荐授权弹窗**（含知情同意文案，方案 10.4 节有示例）
- 需在设置页增加"智能推荐"**开关**（关闭后回退 V1 轮播）
- 需增加"清除 AI 推荐数据"按钮

### Pad 端团队（相框端）

- **无需改动**，API 接口完全兼容 V1

### 法务 / 合规团队

- 需审核 AI 推荐功能的**隐私政策更新文案**
- 需与 AI 服务商签订 **DPA 协议**（方案 10.5 节有条款清单）
- 需完成**个人信息保护影响评估（PIA）**
- 确认所选 AI 模型数据存储在中国境内

### 测试团队

- 需验证数据脱敏逻辑（抓包确认传给 AI 的数据不含真实 ID、姓名、精确位置）
- 需验证用户授权 → 关闭 → 回退 V1 的完整流程
- 需覆盖 AI 降级场景（API 超时、格式异常、余额不足）
- A/B 测试：V1 vs V2，按 userId 各 50%，周期 14 天
