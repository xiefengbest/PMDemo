# 智能精选算法技术文档

本文档详细描述「本地照片智能精选」算法的设计、实现与优化，适用于从连拍或相似照片中自动选出最优照片的场景。**基于当前代码逻辑编写**。

---

## 一、算法概述

### 1.1 目标

从本地相册的**连拍或相似照片**中，自动筛选出**合格的最优照片**（仅质量过滤，无数量上限），过滤掉重复、模糊或质量较差的照片，减少用户手动筛选的工作量。

### 1.2 核心策略

1. **时间分组**：将连续拍摄的照片（≤5 秒，同一天）归为一组；无 dateTaken 时使用 500ms 窗口避免批量导入误分组
2. **相似度去重**：使用 dHash 算法（汉明距离 ≤12），并查集聚类，簇内选综合分最高的一张作为代表
3. **质量评分**：基于 Laplacian + Tenengrad 清晰度、人脸检测，综合分 finalScore（base + normLap/normTen + 绝对清晰度）
4. **最优选择**：Tenengrad 决定是否丢弃；finalScore 超门槛则合格，合格的全部保留，否则组内保留综合分最高 1 张

### 1.3 算法特点

- **稳定性**：相同输入多次执行结果一致（二级排序、稳定比较器）
- **高效性**：并行处理、缩略图优化、O(n) 时间分组
- **准确性**：结合时间、相似度、清晰度、人脸多维度判断

## 二、完整流程

### 2.1 全量精选管道（SmartSelectPipeline.process）

```
输入：照片列表（未排序）
    ↓
┌─────────────────────────────────────────────────────────┐
│ 步骤 1：按拍摄时间排序                                    │
│ - 优先使用 dateTaken，否则使用 dateAdded                  │
│ - 二级排序：compareBy({ getSortTime() }, { generateId() })│
└─────────────────────────────────────────────────────────┘
    ↓
┌─────────────────────────────────────────────────────────┐
│ 步骤 2：时间窗口分组（TimeGrouping.groupByBurstTime）     │
│ - 时间差 ≤ 5秒 且 同一天 → 归入同一连拍组                 │
│ - 双方无 dateTaken 时窗口 500ms                           │
│ - 单张照片独立成组                                        │
└─────────────────────────────────────────────────────────┘
    ↓
┌─────────────────────────────────────────────────────────┐
│ 步骤 3：对每个连拍组并行处理（async + awaitAll）          │
│                                                          │
│  3.1 计算每张照片的 dHash（64-bit）                      │
│      - 生成灰度缩略图（ThumbnailGenerator）              │
│      - Resize 到 9×8（DHashCalculator）                  │
│      - 计算横向差值 → 64-bit hash                        │
│                                                          │
│  3.2 相似簇（并查集）：汉明距离 ≤ 12 两两 union          │
│      - 每个簇用 selectBestPhoto 选综合分最高 1 张为代表   │
│      - 非代表 id 进入 similarPhotoIds                    │
│                                                          │
│  3.3 对簇代表做质量过滤（BestPhotoSelector.selectBestPhotos）│
│      - Laplacian + Tenengrad + 人脸；finalScore 公式     │
│      - Tenengrad ≥ 门槛且 finalScore > 门槛 → 合格全留   │
│      - 否则组内保留综合分最高 1 张                       │
└─────────────────────────────────────────────────────────┘
    ↓
┌─────────────────────────────────────────────────────────┐
│ 步骤 4：汇总各组的 selected、merged、finalScores         │
│         按综合分降序、再按时间+id 排序后输出             │
└─────────────────────────────────────────────────────────┘
    ↓

```

## 三、核心组件详解

### 3.1 TimeGrouping（连拍时间分组器）

**文件**：`TimeGrouping.kt`

**规则**：
- `BURST_TIME_WINDOW_MS = 5000L`：有 dateTaken 时，时间差 ≤ 5 秒且同一天归入同一组
- `DATE_ADDED_WINDOW_MS = 500L`：双方都无 dateTaken 时，窗口 500ms，避免批量导入误分组
- 排序：`sortedWith(compareBy({ it.getSortTime() }, { it.generateId() }))`，保证稳定性

**输出**：`List<BurstGroup>`，每个 BurstGroup 含 photos、startTime、endTime

### 3.2 DHashCalculator（dHash 计算器）

**文件**：`DHashCalculator.kt`

**步骤**：
1. 将灰度图 resize 到 9×8
2. 对每一行计算相邻像素差值（9 像素 → 8 差值）
3. 输出 64-bit Long

**支持格式**：RGB_565、ARGB_8888（通过 getGrayValue 自动检测）

### 3.3 SimilarityChecker（相似度判断）

**文件**：`SimilarityChecker.kt`

```kotlin
fun isSimilar(hash1: Long, hash2: Long): Boolean {
    val distance = calculateHammingDistance(hash1, hash2)  // xor + Long.bitCount
    return distance <= HAMMING_THRESHOLD  // 12
}
```

**阈值**：汉明距离 ≤ 12（HAMMING_THRESHOLD）视为相似。管道内用并查集对组内照片建簇，簇内选综合分最高的一张为代表。

### 3.4 BestPhotoSelector（组内选择器）

**文件**：`BestPhotoSelector.kt`

**selectBestPhoto**（簇内选代表）：对给定组内照片计算 Laplacian + Tenengrad + 人脸，得到 finalScore，保留**综合分最高的一张**。用于相似簇代表选择。

**selectBestPhotos 流程**（对簇代表做质量筛选）：

1. 并行计算每张照片的清晰度（Laplacian + Tenengrad）和人脸
2. baseScore = 归一化清晰度×0.7 + (有人脸 ? 1.0 : 0.0)×0.3
3. finalScore = base×WB + normLap×WL + normTen×WT + absLap×WAL + absTen×WAT（小组时 normLap/normTen 压缩到 [0.5, 0.7]）
4. 合格：Tenengrad ≥ TENENGRAD_FLOOR(900) 且 finalScore > 门槛（小组 0.25，否则 0.4）
5. 保留规则：有合格则合格全留；仅 Tenengrad 达标则留综合分最高 1 张；无人 Tenengrad 达标则留综合分最高 1 张
6. 稳定排序：按 finalScore 降序、时间、generateId()

### 3.5 SmartSelectPipeline（精选管道）

**文件**：`SmartSelectPipeline.kt`

**processGroup 流程**（单组处理）：
1. 单张照片直接返回
2. 并行计算每张照片的 dHash
3. **并查集聚类**：两两 isSimilar(hash1, hash2)（汉明≤12）则 union，得到多个簇
4. **簇内代表**：每个簇 1 张则直接为代表；多张则 `selectBestPhoto(clusterGroup)` 选综合分最高 1 张；非代表 id 为 similarPhotoIds
5. 对「各簇代表」组成的 deduplicatedGroup 调用 `selectBestPhotos(deduplicatedGroup)`
6. 合并 similarPhotoIds + selectionResult.mergedPhotoIds 作为 mergedPhotoIds；最终列表按综合分降序、再按时间+id 排序

**输出**：`SmartSelectResult(selectedPhotos, mergedPhotoIds, groupStats)`

## 五、扫描流程与双次处理

### 5.1 扫描逻辑（scanAndRecognizePhotos）

- 分页扫描：`photoScanner.scanPhotosPaginated(pageSize)`，按 DATE_TAKEN DESC
- 批次合并到 `allPhotosByMonth`，按月份聚合

### 5.2 双次处理机制

**第一次（快速处理）**：
- 条件：`!firstMonthProcessed && firstMonthData.size >= minSizeForQuickProcess`（30 张）
- 行为：用当前累积数据调用 `processMonthPhotos`
- **不加入 processedMonths**，以便后续用完整数据再处理

**第二次（完整处理）**：
- 条件：`newestMonthInBatch != lastSeenMonth`（出现新月份）
- 行为：用 `allPhotosByMonth[previousMonth]` 完整数据调用 `processMonthPhotos`
- 加入 processedMonths

**结果**：第一个月份会被处理两次（部分数据 → 完整数据）

### 5.3 UI 优化（避免闪烁）

**updateMonthGroup**：当 `existing.isSmartSelectDone` 时，保留 `existing.recognizedCount`，不覆盖为 0

**updateMonthSmartSelectStart**：当 `existing.isSmartSelectDone` 时，保留 `existing.recognizedCount`，不覆盖为 photoCount

**loadCoverPhoto**：当 `validPhotoCount > existing.recognizedCount` 时，查询 `hasSkipUploadPhotos`；若 > 0 则更新，否则跳过（避免 clearSkipUpload 中间态覆盖）

## 六、参数配置

### 6.1 SmartSelectConfig

| 参数 | 值 | 说明 |
|------|---|------|
| NORMALIZED_SHARPNESS_FLOOR | 0.2 | 组内 Laplacian 归一化底边 |
| NORMALIZED_LAP_TEN_FLOOR | 0.2 | normLap/normTen 底边（常规组） |
| TENENGRAD_FLOOR | 900 | Tenengrad 阈值，低于视为模糊/假清晰 |
| FINAL_SCORE_FLOOR | 0.4 | 综合分门槛（常规组） |
| SMALL_GROUP_THRESHOLD | 2 | 组内张数 ≤ 此值视为小组 |
| NORMALIZED_LAP_TEN_FLOOR_SMALL_GROUP | 0.5 | 小组 normLap/normTen 底边 |
| NORMALIZED_LAP_TEN_CEIL_SMALL_GROUP | 0.7 | 小组 normLap/normTen 顶边 |
| FINAL_SCORE_FLOOR_SMALL_GROUP | 0.25 | 小组综合分门槛 |
| LAP_ABS_MAX / TEN_ABS_MAX | 3000 / 20000 | 绝对清晰度映射上限 |
| WEIGHT_BASE_SCORE 等 | 0.4, 0.2, 0.15, 0.15, 0.1 | finalScore 各分量权重 |

## 八、注意事项

### 8.1 稳定性

- 排序均使用 `generateId()` 作为二级排序键
- BestPhotoSelector 使用 `thenBy { it.photo.generateId() }` 保证得分相同时结果一致

### 8.2 SQLite 限制

- `getPhotosByIds` 单次查询 id 数量需 ≤ 500（SQLITE_MAX_VARIABLE_NUMBER），超出需分批

### 8.3 资源释放

- 使用完毕后调用 `pipeline.close()` 释放 FaceDetector
- Bitmap 使用后调用 `recycle()`
