# 月份详情页（批量发送）— 交互产品说明

> 页面标识：`MonthDetailActivity`
> 最后更新：2026-02-26

---

## 一、页面概述

用户从「照片分析」页面点击某个月份卡片后进入本页面。页面展示该月份经过 AI 智能整理后的照片，用户在此完成**选图 → 发送到相框**的核心操作。

**设计理念**：默认全选、一键发送。用户只需取消不想要的照片，而非从零开始挑选。

---

│  │ ☑ ○ │ │ ☑ ○ │ │ ☑ ○ │      │  ← 3列照片宫格

│  └─────┘ └─────┘ └─────┘      │

### 区域说明

| 区域 | 元素 | 说明 |
|------|------|------|
| 顶部导航栏 | ← 返回按钮 | 点击返回上一页（照片分析页） |
| | 标题「批量发送」 | 固定文字 |
| | 「发送(N)」按钮 | 显示当前选中数量，点击发送选中照片 |
| 月份信息区 | 月份标题 | 如「2024年3月」或日期跨度格式 |
| | 照片计数 | 如「共 25 张照片」，显示当前页面总照片数 |
| 照片宫格区 | 3列网格 | 1:1 正方形卡片，圆角 8dp，间距 4dp |
| | 勾选图标 | 每张照片右上角，40dp 点击热区 |
| 底部操作栏 | 「全选」/「取消全选」 | 切换全选状态 |
| | 「删除」 | 批量删除选中照片 |

## 三、核心交互流程

### 3.1 进入页面（初始状态）

```
用户点击月份卡片
    │
    ▼
页面打开，加载该月份照片
    │
    ├── 正在加载中 → 显示「正在精选，请稍候」加载态
    │
    ▼
照片加载完成
    │
    ▼
所有照片默认全选（每张照片右上角显示已勾选图标）
    │
    ▼
发送按钮显示「发送(N)」（N = 照片总数）
底部栏显示「取消全选」（因为当前是全选状态）
```

**关键规则**：
- 仅首次加载时自动全选，后续数据更新（如删除照片后）保持用户当前选择状态
- 照片无选中蒙版遮罩，始终清晰展示原图

---

### 3.2 照片选择操作

#### 点击照片区域 → 进入大图详情

```
用户点击照片图片区域
    │
    ▼
跳转到 PhotoDetailActivity（大图详情页）
    │
    ├── 可左右滑动浏览同月份所有照片
    ├── 可在详情页中删除照片
    │
    ▼
返回本页面
    │
    ├── 如果在详情页有删除操作 → 自动刷新照片列表
    └── 选中状态保持不变
```

#### 点击勾选图标 → 切换选中状态

```
用户点击照片右上角勾选图标（40dp 热区）
    │
    ├── 当前已选中 → 取消选中（图标变为未选中样式）
    └── 当前未选中 → 选中（图标变为已选中样式）
    │
    ▼
同步更新：
  - 顶部「发送(N)」按钮的数量
  - 底部「全选/取消全选」按钮文字
  - 如果选中数为 0：发送按钮显示「发送」，置灰不可点击
```

#### 点击「全选/取消全选」

```
用户点击底部栏左侧按钮
    │
    ├── 当前非全选状态 → 按钮文字「全选」
    │   点击后：所有照片选中，按钮变为「取消全选」
    │
    └── 当前全选状态 → 按钮文字「取消全选」
        点击后：所有照片取消选中，按钮变为「全选」
    │
    ▼
同步更新发送按钮数量和状态
```

---

### 3.3 发送流程

```
用户点击「发送(N)」
    │
    ├── 选中数 = 0 → Toast「请先选择要发送的照片」，流程结束
    │
    ▼
检查相框绑定状态
    │
    ├── 未绑定相框 ──────────────────────┐
    │                                    ▼
    │                           显示「绑定相框」弹窗
    │                                    │
    │                            ┌───────┴───────┐
    │                            ▼               ▼
    │                        点击「绑定」      点击「取消」
    │                            │              → 流程结束
    │                            ▼
    │                      跳转扫码页面
    │                            │
    │                            ▼
    │                      扫码绑定成功
    │                            │
    │                            ▼
    │                  自动获取相框列表，继续发送流程 ──┐
    │                                                │
    ├── 已绑定相框 ──────────────────────────────────┤
    │                                                │
    ▼                                                ▼
                    显示「选择相框」弹窗
                    （列出所有已绑定的相框设备）
                             │
                     ┌───────┴───────┐
                     ▼               ▼
                 点击「确定」      点击「取消」
                     │              → 流程结束
                     ▼
              开始上传照片
                     │
                     ├── 屏幕保持常亮，防止中断
                     ├── 显示上传进度弹窗（已发布数/总数）
                     ├── 分批上传，每批完成后更新进度
                     │
                     ▼
              ┌──────┴──────┐
              ▼             ▼
          上传成功        上传失败
              │             │
              ▼             ▼
      显示成功弹窗      Toast 提示
      （发送成功！）    （根据网络状态
              │          显示不同文案）
              ▼
      弹窗关闭后
      自动返回照片分析页
      并刷新月份回顾数据
```

**上传过程保护机制**：
- 屏幕常亮（`FLAG_KEEP_SCREEN_ON`）
- CPU 唤醒锁（`WakeLock`，最长 1 小时超时）
- 上传完成/失败后自动释放

---

### 3.4 删除流程

#### 批量删除（底部栏「删除」按钮）

```
用户点击底部栏「删除」
    │
    ├── 选中数 = 0 → Toast「请先选择照片」，流程结束
    │
    ▼
显示确认弹窗
「确认删除 N 张照片？」
    │
    ├── 点击「取消」→ 流程结束
    │
    ▼
点击「删除」
    │
    ▼
执行删除（标记为已删除，非物理删除）
    │
    ▼
Toast「已删除 N 张照片」
    │
    ▼
照片列表自动刷新
    │
    ├── 仍有剩余照片 → 正常显示，选中状态保持
    │
    └── 所有照片已删除 → 自动关闭页面，返回照片分析页
```

#### 从详情页删除（单张）

```
用户在大图详情页删除某张照片
    │
    ▼
返回本页面
    │
    ▼
照片列表自动刷新（已删除的照片消失）
    │
    ├── 仍有剩余照片 → 正常显示
    └── 所有照片已删除 → 自动关闭页面
```

## 四、状态说明

### 4.1 发送按钮状态

| 选中数量 | 按钮文字 | 可点击 | 透明度 |
|---------|---------|--------|--------|
| 0 | 「发送」 | 否 | 0.5（置灰） |
| N > 0 | 「发送(N)」 | 是 | 1.0（正常） |

### 4.2 全选按钮状态

| 条件 | 按钮文字 |
|------|---------|
| 所有照片均已选中 | 「取消全选」 |
| 存在未选中的照片 | 「全选」 |

### 4.3 照片卡片状态

| 状态 | 勾选图标 | 照片显示 | 删除按钮 |
|------|---------|---------|---------|
| 已选中 | ☑ 已勾选样式 | 原图清晰展示（无蒙版） | 隐藏 |
| 未选中 | ○ 未勾选样式 | 原图清晰展示（无蒙版） | 隐藏 |

### 4.4 页面自动关闭条件

当满足以下条件时页面自动关闭并返回上一级：
- 之前有照片（`previousPhotoCount > 0`）且当前照片数变为 0
- 批量删除后剩余照片数为 0
- 单张删除后剩余照片数为 0
- 发送成功弹窗关闭后

## 五、边界情况处理

| 场景 | 处理方式 |
|------|---------|
| 照片正在精选中 | 显示「正在精选，请稍候」加载态，照片列表为空 |
| 未登录点击发送 | Toast「请先登录」 |
| 网络异常上传失败 | Toast「请检查网络连接后重试」 |
| 服务器异常上传失败 | Toast「上传失败，请重试」 |
| 检查相框绑定失败 | Toast 显示具体错误信息 |
| 页面已销毁时回调 | 跳过 UI 操作，避免崩溃 |
| 初始加载照片为空 | 不触发自动关闭（仅当从非空变为空时才关闭） |

## 六、涉及的页面跳转

```
PhotoAnalysisFragment（照片分析页）
    │
    │  点击月份卡片
    ▼
MonthDetailActivity（本页面 — 批量发送）
    │
    ├── 点击照片 → PhotoDetailActivity（大图详情页）
    ├── 点击发送 → ChooseFrameDialog（选择相框弹窗）
    ├── 未绑定相框 → FrameBindDialog → QrCodeScanActivity（扫码绑定）
    ├── 发送成功 → FrameOperationSuccessDialog（成功弹窗）→ 自动返回
    └── ← 返回 → PhotoAnalysisFragment
```

## 七、技术备注

| 项目 | 说明 |
|------|------|
| 照片网格 | RecyclerView + GridLayoutManager（3列） |
| 照片卡片 | CardView，1:1 比例，圆角 8dp |
| 数据源 | Room 数据库 Flow，实时响应增删 |
| 图片加载 | Glide，0.1 倍缩略图 + centerCrop |
| 勾选图标 | 40dp 点击热区（视觉 24dp + 8dp padding），elevation 4dp |
| 上传方式 | 分批上传 + 分批发布，支持断点标记 |
| 删除方式 | 逻辑删除（`isDeleted = true`），数据库查询自动过滤 |
