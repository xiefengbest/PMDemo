# 批量照片上传优化方案

## 一、问题背景

### 1.1 当前场景
用户从手机相册选择照片发送到相框：
- 照片按月分组显示
- 用户点击"发送到相框"后一键发送该月所有照片
- 每10张照片为一批，先上传到OSS，再调用发布接口

### 1.2 现存问题

| 问题 | 影响 | 严重程度 |
|------|------|----------|
| 串行上传 | 500+张照片需要15-25分钟 | ⭐⭐⭐⭐⭐ |
| 无防息屏机制 | 息屏后CPU休眠，上传中断 | ⭐⭐⭐⭐⭐ |
| 无断点续传 | 中断后需要重新上传 | ⭐⭐⭐ |
| 用户体验差 | 息屏后状态异常，已发布的照片看不到 | ⭐⭐⭐⭐ |

### 1.3 代码分析

**当前上传流程** (`UploadImageTask.kt`):

## 三、短期优化实施细节

### 3.1 并发上传实现

#### 修改文件：`UploadImageTask.kt`

**修改前**：串行上传
```kotlin
private fun uploadBatch(
    batchImgList: List<SelectImgBean>,
    batchIndex: Int,
    totalBatches: Int
): Boolean {
    for (i in batchImgList.indices) {
        val bean = batchImgList[i]
        uploadSingleImage(bean)  // 串行
    }
}
```

**修改后**：并发上传
```kotlin
private suspend fun uploadBatch(
    batchImgList: List<SelectImgBean>,
    batchIndex: Int,
    totalBatches: Int
): Boolean {
    // 使用信号量控制并发数，与OSS配置保持一致
    val semaphore = Semaphore(CONCURRENT_UPLOAD_COUNT)
    
    val uploadJobs = batchImgList.map { bean ->
        coroutineScope.async(Dispatchers.IO) {
            semaphore.withPermit {
                if (!shouldUploadImage(bean)) {
                    return@async true
                }
                uploadSingleImage(bean)
                bean.uploadsuccess == 1
            }
        }
    }
    
    // 等待所有上传完成
    val results = uploadJobs.awaitAll()
    return results.all { it }
}

companion object {
    const val CONCURRENT_UPLOAD_COUNT = 5  // 并发上传数量
}
```

#### 修改要点：
1. `uploadAllImages()` 方法改为 suspend 函数
2. `uploadBatch()` 方法改为 suspend 函数，内部使用协程并发
3. 使用 `Semaphore` 控制并发数量，避免同时上传过多导致内存溢出
4. 进度更新逻辑需要适配并发场景

### 3.2 屏幕常亮 + CPU唤醒

#### 修改文件：`MonthDetailActivity.kt`

```kotlin
class MonthDetailActivity : BaseActivity() {
    
    // WakeLock 用于防止CPU休眠
    private var wakeLock: PowerManager.WakeLock? = null
    
    /**
     * 开始上传时调用：保持屏幕常亮 + CPU唤醒
     */
    private fun acquireWakeLock() {
        // 1. 保持屏幕常亮
        window.addFlags(WindowManager.LayoutParams.FLAG_KEEP_SCREEN_ON)
        
        // 2. 获取WakeLock防止CPU休眠
        if (wakeLock == null) {
            val powerManager = getSystemService(Context.POWER_SERVICE) as PowerManager
            wakeLock = powerManager.newWakeLock(
                PowerManager.PARTIAL_WAKE_LOCK,
                "HKPhotoAlbum:UploadWakeLock"
            )
        }
        wakeLock?.acquire(60 * 60 * 1000L)  // 最长1小时，实际会在完成后释放
        
        Log.d(TAG, "WakeLock acquired, 屏幕常亮已开启")
    }
    
    /**
     * 上传完成/失败/取消时调用：恢复正常息屏行为
     */
    private fun releaseWakeLock() {
        // 1. 移除屏幕常亮标志，恢复自动息屏
        window.clearFlags(WindowManager.LayoutParams.FLAG_KEEP_SCREEN_ON)
        
        // 2. 释放WakeLock
        wakeLock?.let {
            if (it.isHeld) {
                it.release()
            }
        }
        wakeLock = null
        
        Log.d(TAG, "WakeLock released, 已恢复正常息屏")
    }
```

## 六、注意事项

### 6.1 WakeLock 使用注意

1. **必须配对使用**：`acquire()` 和 `release()` 必须成对调用
2. **设置超时时间**：防止异常情况下永久持有
3. **在 onDestroy 中释放**：防止 Activity 异常销毁时泄漏
4. **使用 try-finally**：确保异常情况下也能释放

### 6.2 并发上传注意

1. **控制并发数**：不要超过5个，避免内存溢出
2. **进度更新线程安全**：使用 AtomicInteger 或 synchronized
3. **错误处理**：单个图片失败不应影响其他图片上传
4. **取消处理**：支持用户取消上传时正确中断所有协程

### 6.3 用户体验注意

1. **明确提示**：告知用户上传过程中请勿离开
2. **进度显示**：实时显示已上传/总数
3. **完成反馈**：上传完成后给予明确提示
4. **失败重试**：提供重试入口
