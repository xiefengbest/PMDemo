# 合辑功能增强 - 产品需求文档

## 一、功能概述

| 模块 | 功能 | 入口位置 | 说明 |
|------|------|----------|------|
| 人像合辑 | 隐藏 | 详情页右上角菜单 | 隐藏后该人像不再出现在首页 |
| 人像合辑 | 合并 | 详情页右上角菜单 | 将当前人物合并到另一个人物 |
| 月份合辑 | 删除 | 详情页右上角菜单 | 删除合辑及其内所有照片 |
| 设置页 | 重置人物建议 | 设置页 | 恢复已隐藏人物，重新识别照片 |

---

## 二、人像合辑 - 隐藏功能

### 2.1 功能说明

- **入口**：人像合辑详情页 → 右上角更多菜单 → 「隐藏」
- **效果**：隐藏后，该人像将不再显示在首页的人物列表中
- **恢复方式**：通过「设置」→「重置人物建议」可批量恢复所有已隐藏的人物

### 2.2 交互流程

```
人像详情页 → 点击更多菜单 → 点击「隐藏」
    ↓
弹出确认对话框：
    ┌─────────────────────────────────────┐
    │             隐藏确认                 │
    ├─────────────────────────────────────┤
    │                                     │
    │  确定要隐藏此人物吗？                 │
    │  隐藏后该人物将不再显示在首页。        │
    │                                     │
    ├─────────────────────────────────────┤
    │       [取消]           [确定]        │
    └─────────────────────────────────────┘
    ↓
确认后 → 调用隐藏接口 → 成功后自动返回首页
```

### 2.3 API 接口

> 🔶 **当前为模拟接口，待服务端实现**

| 项目 | 内容 |
|------|------|
| 路径 | `POST /v1/pc/face/hide` |
| 功能 | 隐藏人物（仅隐藏，不支持取消隐藏） |

**请求参数：**

| 参数 | 类型 | 必填 | 说明 |
|------|------|------|------|
| userId | String | 是 | 用户ID |
| token | String | 是 | 用户Token |
| faceId | String | 是 | 人物ID |

**返回结果：** 标准 SimpleResultBody

## 三、人像合辑 - 合并功能

### 3.1 功能说明

- **入口**：人像合辑详情页 → 右上角更多菜单 → 「合并」
- **效果**：将当前人物合并到另一个人物，当前人物消失，其照片归入目标人物
- **场景**：同一个人被识别为多个人物时，可通过此功能合并

### 3.2 交互流程

```
人像详情页 → 点击更多菜单 → 点击「合并」
    ↓
弹出人物选择对话框：
    ┌─────────────────────────────────────┐
    │            选择人物                  │
    ├─────────────────────────────────────┤
    │  ○ [头像] 张三     12张              │
    │  ○ [头像] 李四     8张               │
    │  ○ [头像] 王五     5张               │
    │  ...                                │
    ├─────────────────────────────────────┤
    │       [取消]          [确认]         │
    └─────────────────────────────────────┘
    ↓
选择目标人物并确认 → 调用合并接口 → 成功后自动返回首页
```

### 3.3 API 接口

> 🔶 **当前为模拟接口，待服务端实现**

| 项目 | 内容 |
|------|------|
| 路径 | `POST /v1/pc/face/merge` |
| 功能 | 将源人物合并到目标人物 |

**请求参数：**

| 参数 | 类型 | 必填 | 说明 |
|------|------|------|------|
| userId | String | 是 | 用户ID |
| token | String | 是 | 用户Token |
| sourceFaceId | String | 是 | 源人物ID（被合并的人物，将消失） |
| targetFaceId | String | 是 | 目标人物ID（合并到的人物，将保留） |

**返回结果：** 标准 SimpleResultBody

**合并逻辑说明：**
- 合并后，`sourceFaceId` 对应的人物将不再存在
- `sourceFaceId` 人物下的所有照片将归入 `targetFaceId` 人物
- `targetFaceId` 人物的照片数量 = 原照片数 + sourceFaceId 的照片数

## 四、月份合辑 - 删除功能

### 4.1 功能说明

- **入口**：月份合辑详情页 → 右上角更多菜单 → 「删除合辑」
- **效果**：删除该月份合辑，**同时删除合辑内的所有照片**
- **⚠️ 注意**：此操作不可逆，照片将被永久删除

### 4.2 交互流程

```
月份详情页 → 点击更多菜单 → 点击「删除合辑」
    ↓
弹出确认对话框：
    ┌─────────────────────────────────────┐
    │             删除确认                 │
    ├─────────────────────────────────────┤
    │                                     │
    │  确定要删除此合辑吗？                 │
    │                                     │
    │  ⚠️ 删除后，合辑内的所有照片将被      │
    │  永久删除，此操作不可恢复。           │
    │                                     │
    ├─────────────────────────────────────┤
    │       [取消]           [删除]        │
    └─────────────────────────────────────┘
    ↓
确认后 → 调用删除接口 → 成功后自动返回首页
```

### 4.3 API 接口

> 🔶 **当前为模拟接口，待服务端实现**

| 项目 | 内容 |
|------|------|
| 路径 | `POST /v1/photoFrame/group/delete` |
| 功能 | 删除月份合辑及其内所有照片 |

**请求参数：**

| 参数 | 类型 | 必填 | 说明 |
|------|------|------|------|
| userId | String | 是 | 用户ID |
| token | String | 是 | 用户Token |
| monthTimestamp | Long | 是 | 月份时间戳 |

**返回结果：** 标准 SimpleResultBody

## 五、设置页 - 重置人物建议

### 5.1 功能说明

- **入口**：设置页 → 「重置人物建议」
- **效果**：
  1. 恢复所有已隐藏的人物（重新显示在首页）
  2. 取消所有已合并的人物（恢复为独立的人物）
  3. 对未识别的照片重新进行人脸识别
- **场景**：用户想要找回之前隐藏的人物、恢复已合并的人物，或者希望重新识别照片中的人脸

### 5.2 交互流程

```
设置页 → 点击「重置人物建议」
    ↓
弹出确认对话框：
    ┌─────────────────────────────────────┐
    │           重置人物建议               │
    ├─────────────────────────────────────┤
    │                                     │
    │  确定要重置人物建议吗？               │
    │                                     │
    │  重置后将：                          │
    │  1. 恢复所有已隐藏的人物              │
    │  2. 取消所有已合并的人物              │
    │  3. 重新识别照片                     │
    │                                     │
    ├─────────────────────────────────────┤
    │       [取消]           [确定]        │
    └─────────────────────────────────────┘
    ↓
确认后 → 调用重置接口 → 成功后提示「重置成功」
```

### 5.3 API 接口

> 🔶 **当前为模拟接口，待服务端实现**

| 项目 | 内容 |
|------|------|
| 路径 | `POST /v1/pc/face/reset` |
| 功能 | 重置人物建议（恢复隐藏人物 + 取消合并 + 重新识别） |

**请求参数：**

| 参数 | 类型 | 必填 | 说明 |
|------|------|------|------|
| userId | String | 是 | 用户ID |
| token | String | 是 | 用户Token |

**返回结果：** 标准 SimpleResultBody

## 八、注意事项

1. **Debug 模式限制**
   - 所有新增功能入口（隐藏、合并、删除、重置人物建议）仅在 Debug 模式下显示
   - Release 包不会显示这些功能入口
   - 接口上线后，移除 `BuildConfig.DEBUG` 判断即可启用

2. **隐藏 vs 删除**
   - 人像合辑只能「隐藏」，不能删除（避免误删人脸数据）
   - 月份合辑只能「删除」，不能隐藏（删除时会同步删除照片）

3. **隐藏的恢复**
   - 单个隐藏的人物无法单独恢复
   - 需要通过「重置人物建议」批量恢复所有已隐藏人物

4. **合并的不可逆性**
   - 人物合并后无法拆分
   - 合并前应提示用户确认

5. **删除的不可逆性**
   - 月份合辑删除后，其内照片将被永久删除
   - 删除前必须明确提示用户
