# 批量上传推送优化方案

本文档描述了 Mobile 端批量上传照片时，相框端显示待下载数量的优化方案。

---

## 一、问题背景

### 1.1 现状

- Mobile 端上传照片时，由于 POST 限制，服务端限制每次最多上传 **10 张**
- 相框端接收到推送通知后，每 10 张收到一个 push
- 如果 Mobile 传 100 张，相框接收到的待下载数量会不断变化：10 → 20 → 30 → ... → 100

### 1.2 问题

用户体验差：相框端显示的待下载数量一直跳动，让用户困惑。

### 1.3 期望效果

- Mobile 上传 100 张照片时，相框端从一开始就显示"待下载：100 张"
- 数字稳定，不跳动
- 支持多个上传任务同时进行时正确累加

---

## 二、解决方案

### 2.1 核心设计

**Mobile 端上传时带两个字段**：

| 字段 | 类型 | 说明 |
|-----|------|------|
| `batchId` | Long | 批次号（当前时间戳），同一次上传的所有批次使用相同的 batchId |
| `displayTotal` | Int | 本批次发布总数（仅用于显示） |

### 2.2 字段详细说明

#### batchId（批次号）

| 属性 | 说明 |
|-----|------|
| **类型** | Long（64位整数） |
| **生成方式** | `System.currentTimeMillis()`，即当前时间戳（毫秒） |
| **示例值** | `1706500000000` |
| **作用** | 标识同一次上传任务，相框端用于去重 |
| **生命周期** | 每次用户触发上传/同步操作时生成新的 batchId |

**示例**：
```
用户点击"发送到相框"按钮
  └── 生成 batchId = 1706500000000（当前时间戳）
      ├── 第1批上传 → batchId = 1706500000000
      ├── 第2批上传 → batchId = 1706500000000（相同）
      └── 第3批上传 → batchId = 1706500000000（相同）

用户再次点击"发送到相框"按钮
  └── 生成 batchId = 1706500060000（新的时间戳）
```

#### displayTotal（显示总数）

| 属性 | 说明 |
|-----|------|
| **类型** | Int（32位整数） |
| **取值** | 本次上传的总图片数量，或 0（表示不传/未知） |
| **示例值** | `100`、`50`、`0` |
| **作用** | 相框端显示"待下载：XX张"，避免数字跳动 |
| **特殊值** | 当值为 `0` 时，相框端按实际收到的数量累加 |

**示例**：
```json
// 上传 100 张照片，分 10 批
{
    "batchId": 1706500000000,
    "displayTotal": 100,        // 每批都传相同的 displayTotal
    "imageList": [...]          // 当前批次的 10 张照片
}

// 同步整个合辑（不知道具体数量）
{
    "batchId": 1706500060000,
    "displayTotal": 0,          // 或不传此字段
    "fromAlbumId": 123
}

// 多选发送 30 张照片
{
    "batchId": 1706500120000,
    "displayTotal": 30,         // 选中的照片数量
    "fromImageIds": "1,2,3..."
}
```

### 2.3 相框端显示逻辑

- **相同 batchId**：`displayTotal` 不累加（同一次上传的多批推送）
- **不同 batchId**：`displayTotal` 累加（不同次上传）
- **无 batchId 或 batchId=0**：兼容旧版本，按实际收到的数量累加

### 2.4 流程示意

```
Mobile 上传 100 张照片 (batchId=1706500000000, displayTotal=100)
│
├── 第1批上传 10 张 ──→ 服务端 ──→ 推送给相框
│                                   └── 相框记录: batchId=1706500000000, displayTotal=100
│                                   └── 显示: "待下载：100张"
│
├── 第2批上传 10 张 ──→ 服务端 ──→ 推送给相框
│                                   └── 相同 batchId，displayTotal 不累加
│                                   └── 显示: "待下载：100张"（不变）
│
├── ... (继续上传)
│
└── 第10批上传 10 张 ──→ 服务端 ──→ 推送给相框
                                    └── 显示: "待下载：100张"（不变）
```

### 2.5 多任务并发场景

```
用户 A 上传 100 张 (batchId=A, displayTotal=100)
用户 B 同时转发 50 张 (batchId=B, displayTotal=50)

相框收到推送：
├── batchId=A, displayTotal=100 → 记录 A:100
├── batchId=B, displayTotal=50  → 记录 B:50
├── batchId=A, displayTotal=100 → 已存在，不累加
└── ...

显示总数 = 100 + 50 = 150 张
```

## 三、触发场景说明

### 3.1 什么情况会生成新的 batchId

每次用户触发一个新的上传任务，就会生成一个新的 `batchId`（时间戳）。

| 触发场景 | 入口文件 | 说明 |
|---------|---------|------|
| **月份详情页发送** | `MonthDetailActivity.kt` | 用户浏览某月照片，选择后发送到相框 |
| **智能识别页发送** | `PhotoAnalysisFragment.kt` | 用户从智能分类中选择照片发送 |
| **照片预览页发送** | `PhotoPreviewFragment.kt` | 用户预览照片时直接发送 |

### 3.2 触发规则

| 情况 | 是否生成新 batchId | 说明 |
|-----|-------------------|------|
| 用户点击"发送到相框" | ✅ 是 | 每次点击都是新的上传任务 |
| 用户从不同页面发起上传 | ✅ 是 | 不同入口，不同任务 |
| 用户先后多次上传 | ✅ 是 | 时间戳不同 |
| 上传失败后重试 | ✅ 是 | 重新创建任务，重新生成 batchId |
| 同一次上传的分批（10张/批） | ❌ 否 | 使用相同 batchId |

### 3.3 场景示例

```
场景1：用户在月份详情页选择 50 张照片发送
└── 生成 batchId = 1706500000000, displayTotal = 50
    ├── 第1批 10 张 → batchId = 1706500000000
    ├── 第2批 10 张 → batchId = 1706500000000
    └── ... (同一 batchId)

场景2：用户又在智能识别页选择 30 张照片发送
└── 生成 batchId = 1706500060000, displayTotal = 30
    ├── 第1批 10 张 → batchId = 1706500060000
    └── ... (同一 batchId，但与场景1不同)

场景3：用户发送完后，再次在月份详情页选择 20 张发送
└── 生成 batchId = 1706500120000, displayTotal = 20
    └── ... (新的 batchId)
```

### 3.4 代码层面

`batchId` 在 `UploadImageTask` 创建时生成，每次用户触发上传都会创建新的 `UploadImageTask` 实例：

```kotlin
class UploadImageTask(uploadBean: UploadBean) {
    // 在创建任务时生成 batchId
    private val batchId = System.currentTimeMillis()
    
    // 后续所有分批上传都使用这个 batchId
}
```

## 四、各端实现

### 4.1 Mobile 端

#### 4.1.1 改动点

在上传照片时，生成 `batchId` 并在每批上传中携带 `batchId` 和 `displayTotal`。

#### 4.1.2 代码示例

**文件**：`feature/mobile/src/main/java/com/hk/phone_album/mobile/upload/UploadImageTask.kt`

```kotlin
class UploadImageTask {
    
    companion object {
        private const val BATCH_SIZE = 10  // 每批上传数量
    }
    
    /**
     * 上传所有图片
     */
    suspend fun uploadAllImages(images: List<SelectImgBean>) {
        // 生成本次上传的批次号（时间戳）
        val batchId = System.currentTimeMillis()
        val displayTotal = images.size
        
        // 分批上传，每批都带相同的 batchId 和 displayTotal
        val batches = images.chunked(BATCH_SIZE)
        for ((index, batch) in batches.withIndex()) {
            val success = uploadBatch(
                photos = batch,
                batchId = batchId,
                displayTotal = displayTotal,
                batchIndex = index,
                totalBatches = batches.size
            )
            if (!success) {
                // 上传失败处理
                break
            }
        }
    }
    
    /**
     * 上传单批图片
     */
    private suspend fun uploadBatch(
        photos: List<SelectImgBean>,
        batchId: Long,
        displayTotal: Int,
        batchIndex: Int,
        totalBatches: Int
    ): Boolean {
        val params = hashMapOf<String, Any?>(
            "userId" to userId,
            "albumId" to albumId,
            "photos" to preparePhotoList(photos),
            "batchId" to batchId,           // 新增：批次号
            "displayTotal" to displayTotal   // 新增：显示总数
        )
        
        return try {
            val result = api.uploadPhotos(params)
            result.isSuccess
        } catch (e: Exception) {
            false
        }
    }
}
```

### 4.3 相框端

#### 4.3.1 改动点

1. 按 `batchId` 分组记录 `displayTotal`，相同 `batchId` 不累加
2. 兼容旧版本（无 `batchId` 或 `batchId=0` 时按实际数量累加）
3. 显示总数 = 新版本批次总和 + 旧版本累计

#### 4.3.2 代码示例

```kotlin
class PhotoPushHandler {
    
    // 新版本：记录每个批次的 displayTotal（batchId -> displayTotal）
    private val batchTotalMap = mutableMapOf<Long, Int>()
    
    // 旧版本兼容：累计实际收到的数量（没有 batchId 或 batchId=0 的推送）
    private var legacyCount = 0
    
    // 实际收到的照片 ID（用于下载）
    private val downloadQueue = mutableListOf<String>()
    
    /**
     * 处理推送消息
     */
    fun onPushReceived(push: PhotoPush) {
        val batchId = push.batchId
        val displayTotal = push.displayTotal
        val photoIds = push.photoIds
        
        // 判断是否为新版本推送（batchId 存在且不为0）
        val isNewVersion = batchId != null && batchId != 0L && displayTotal != null
        
        if (isNewVersion) {
            // ========== 新版本逻辑 ==========
            // 相同 batchId 不重复记录 displayTotal
            if (!batchTotalMap.containsKey(batchId)) {
                batchTotalMap[batchId!!] = displayTotal!!
            }
        } else {
            // ========== 旧版本兼容 ==========
            // 没有 batchId 或 batchId=0，按实际收到的数量累加
            legacyCount += photoIds.size
        }
        
        // 实际照片加入下载队列
        downloadQueue.addAll(photoIds)
        
        // 更新 UI 显示
        updateUI()
        
        // 开始下载照片
        startDownload(photoIds)
    }
    
    /**
     * 更新显示的待下载数量
     */
    private fun updateUI() {
        // 显示总数 = 新版本批次总和 + 旧版本累计
        val newVersionTotal = batchTotalMap.values.sum()
        val total = newVersionTotal + legacyCount
        
        showPendingCount(total)
    }
    
    /**
     * 单张照片下载完成
     */
    fun onPhotoDownloaded(photoId: String) {
        downloadQueue.remove(photoId)
        // 可选：更新下载进度
    }
    
    /**
     * 全部下载完成后重置状态
     */
    fun onAllDownloadComplete() {
        batchTotalMap.clear()
        legacyCount = 0
        downloadQueue.clear()
        updateUI()
    }
    
    /**
     * 清理指定批次（可选：当某个批次全部下载完成时调用）
     */
    fun onBatchComplete(batchId: Long) {
        batchTotalMap.remove(batchId)
        updateUI()
    }
}
```

## 六、异常场景处理

### 6.1 上传中途失败

**场景**：Mobile 上传 100 张，传到第 20 张时失败了。

**表现**：
- 相框显示"待下载：100 张"
- 实际只收到 20 张照片
- 下载完成后，显示的是 100 张，但实际下载了 20 张

**处理建议**：
- 下载完成后可以提示用户实际下载数量
- 或者定期清理超时的批次记录

```kotlin
// 可选：批次超时清理（如 30 分钟无新推送的批次）
fun cleanupStaleBatches() {
    val now = System.currentTimeMillis()
    val staleThreshold = 30 * 60 * 1000  // 30 分钟
    
    batchTotalMap.keys.filter { batchId ->
        now - batchId > staleThreshold
    }.forEach { batchId ->
        batchTotalMap.remove(batchId)
    }
    
    updateUI()
}
```

### 6.2 多设备同时上传

**场景**：手机 A 和手机 B 同时向相框上传照片。

**表现**：
- 手机 A：batchId=A, displayTotal=100
- 手机 B：batchId=B, displayTotal=50
- 相框正确显示 150 张

**无需特殊处理**，方案已支持。
