# 好看智能相框 — 照片发送任务交互规范

> 面向 Android / iOS 原生开发，描述照片发送全流程的交互逻辑、状态流转与 UI 行为规格。

---

## 一、核心设计原则

- **非阻塞**：发送过程不阻断用户操作，用户可继续浏览、选择、发起新任务
- **多任务并行**：支持同时存在多个发送任务（不同相框或多次发送）
- **渐进式反馈**：通过 指示条 → 面板 → 通知 → 弹窗 四层 UI 递进告知用户进度与结果
- **可恢复**：失败照片支持一键重试，无需用户重新选择

---

## 二、任务状态定义与流转

### 状态定义

| 状态 | 含义 | 颜色 |
|---|---|---|
| `sending` | 正在发送中 | 蓝色 `#007AFF` |
| `completed` | 全部发送成功 | 绿色 `#34C759` |
| `partial` | 部分成功、部分失败 | 橙色 `#FF9500` |
| `failed` | 全部失败 | 红色 `#FF3B30` |

### 流转规则

```
                          ┌──→  completed  （failCount == 0）
sending  ── 全部处理完 ──┼──→  partial    （failCount > 0 且 successCount > 0）
                          └──→  failed     （successCount == 0）

partial / failed  ── 用户点击重试 ──→  新建 sending 任务（仅包含失败照片）
completed         ── 8秒后自动清理 ──→  从任务列表移除
```

### 任务数据结构（供参考）

```
Task {
    id:             Int          // 唯一标识
    total:          Int          // 本次发送总张数
    progress:       Int          // 已处理张数（成功 + 失败）
    successCount:   Int          // 成功张数
    failCount:      Int          // 失败张数
    failedPhotoIds: [ID]         // 失败照片标识列表
    targetFrame:    String       // 目标相框名称
    status:         TaskStatus   // 当前状态
    photoIds:       [ID]         // 本次发送的照片标识列表
}
```

## 三、发送主流程

```
用户选择照片 → 选择目标相框 → 确认发送
    │
    ├── 1. 被选照片标记为「发送中」状态（UI 显示发送蒙层）
    ├── 2. 退出多选模式，清空已选列表
    ├── 3. 创建 Task，状态 = sending
    ├── 4. 弹出「开始发送」Banner（见 §5.3）
    └── 5. 开始逐张发送
            │
            │  每张照片：读取原图 → 上传原图 → 记录成功/失败 → 实时更新进度
            │
            └── 全部处理完毕
                 │
                 ├── 判定最终状态（completed / partial / failed）
                 ├── 更新照片状态（成功→已发送，失败→清除发送中标记）
                 ├── 弹出「完成」Banner（见 §5.3）
                 └── 8 秒后自动清理已完成任务
```

### 照片状态标记

照片有两个与发送相关的状态属性：

| 属性 | 含义 | UI 表现 |
|---|---|---|
| `sending` | 正在发送中 | 照片缩略图显示半透明蒙层 + 发送动画 |
| `sent` | 已成功发送过 | 照片缩略图显示已发送角标 |

```
默认         →  选中发送  →  sending = true
发送成功     →  sending = false, sent = true
发送失败     →  sending = false（可重新选择发送）
```

## 四、重试流程

**入口**：任务面板行内"重试"按钮 / 结果弹窗"重新发送失败照片"按钮

```
用户点击重试
    │
    ├── 1. 从原任务中提取失败照片列表
    ├── 2. 删除原任务
    ├── 3. 以失败照片创建新 Task（status = sending）
    ├── 4. 重新标记这些照片为「发送中」
    └── 5. 立即开始发送
```

## 五、四层进度反馈 UI

### 5.1 任务指示条（Task Indicator Bar）

**位置**：底部 Tab 栏正上方，全局常驻

**显示/隐藏规则**：
- 存在 `sending`、`partial` 或 `failed` 状态的任务时 → 显示（带展开动画）
- 否则 → 隐藏（带收起动画）

**最小点击高度**：≥ 44pt（符合 HIG 规范）

**内容规则**：

| 场景 | 左侧文案 | 进度条颜色 | 右侧 |
|---|---|---|---|
| 单任务发送中（无失败） | "发送到{相框名}" | 蓝色 | 百分比 |
| 单任务发送中（有失败） | "发送到{相框名}" | 橙色 | 百分比 |
| 多任务发送中 | "{N} 个任务发送中" | 蓝色/橙色 | 百分比 |
| 仅剩失败/部分失败任务 | "{N} 张发送失败" | 橙色（满条） | "查看" |

**百分比计算**：所有 sending 任务的 `Σprogress / Σtotal × 100`

**点击行为**：展开任务面板（§5.2）

---

### 5.2 任务面板（Task Panel）

**样式**：底部弹出面板（Bottom Sheet），带半透明遮罩

**列表项**：每个任务一行

| 任务状态 | 图标 | 副标题 | 进度条 | 操作按钮 |
|---|---|---|---|---|
| `sending` | 蓝色渐变发送图标 | "{progress}/{total} · {failCount}张失败 · {pct}%" | 蓝色（有失败则橙色） | — |
| `completed` | 绿色渐变勾选 | "已完成 · {total}张" | — | — |
| `partial` | 橙色渐变警告 | "成功{X}张 · 失败{Y}张" | — | "重试" |
| `failed` | 红色渐变叉号 | "全部失败 · {total}张" | — | "重试" |

**行点击**：
- `sending` 状态 → 无响应
- 其他状态 → 关闭面板，打开结果详情弹窗（§5.4）

**底部按钮**："模拟后台发送效果"
- 显示条件：存在 `sending`、`partial` 或 `failed` 状态的任务
- 点击行为：进入后台模拟（见 §六）

**遮罩点击**：关闭面板

---

### 5.3 Banner 通知（顶部横幅）

**样式**：iOS 风格顶部下滑通知横幅，带应用图标 + 标题 + 描述

#### 发送开始 Banner

| 属性 | 值 |
|---|---|
| 图标 | 蓝色渐变发送图标 |
| 标题 | "后台发送中" |
| 描述 | "{N} 张照片正在发送到{相框名}，可继续操作" |
| 自动消失 | 3.5 秒 |
| 点击行为 | 打开任务面板 |

#### 发送完成 Banner

| 最终状态 | 图标 | 标题 | 描述 | 自动消失 |
|---|---|---|---|---|
| `completed` | 绿色勾选 | "好看智能相框" | "已发送 {N} 张照片到{相框名}" | 4 秒 |
| `failed` | 红色叉号 | "好看智能相框" | "{N} 张照片发送失败" | 4 秒 |
| `partial` | 橙色警告 | "好看智能相框" | "成功{X}张，失败{Y}张" | 4 秒 |

**点击行为**：关闭 Banner，打开对应任务的结果详情弹窗（§5.4）

---

### 5.4 结果详情弹窗（Alert Dialog）

**样式**：iOS Alert 风格居中弹窗

| 最终状态 | 图标 | 标题 | 描述 | 补充说明 | 按钮 |
|---|---|---|---|---|---|
| `completed` | 绿色动画勾选 | "发送成功" | "已成功发送 {N} 张照片到{相框名}" | — | 「好的」 |
| `partial` | 橙色警告三角 | "部分发送成功" | "成功 {X} 张，失败 {Y} 张" | "{Y} 张照片因网络异常未能发送到{相框名}" | 「好的」/「重新发送失败照片」 |
| `failed` | 红色叉号 | "发送失败" | "{N} 张照片均发送失败" | "请检查网络连接后重试" | 「好的」/「重新发送失败照片」 |

**按钮行为**：
- 「好的」→ 关闭弹窗
- 「重新发送失败照片」→ 关闭弹窗，触发重试流程（§四）

## 六、后台发送模拟（演示功能）

> 用于演示"用户将 App 切到后台后，照片仍在继续发送"的体验效果。

### 流程

```
点击「模拟后台发送效果」
    │
    ├── 1. 关闭任务面板
    ├── 2. 全屏深色遮罩覆盖，提示"App 已切换到后台"
    ├── 3. 顶部弹出「后台进度通知栏」，实时刷新进度
    │
    │   ── 所有任务发送完毕 ──
    │
    ├── 4. 隐藏进度通知栏
    ├── 5. 延迟 0.6 秒弹出 Push 风格完成通知
    └── 6. 用户点击通知 / 点击「返回 App」→ 退出模拟，回到主界面
```

### 后台进度通知栏

**样式**：仿 iOS 锁屏通知卡片，每 500ms 刷新

| 场景 | 描述文案 |
|---|---|
| 单任务 | "正在发送 {X}/{Y} 张到{相框名}" |
| 多任务 | "{N} 个任务发送中 · {X}/{Y}" |
| 有失败 | 上述文案后追加 "（{N}张失败）" |

显示进度条和百分比。

### Push 完成通知

| 结果 | 图标 | 文案 |
|---|---|---|
| 全部成功 | 绿色勾选 | "全部发送成功 · 共 {N} 张照片" |
| 全部失败 | 红色叉号 | "全部发送失败 · 共 {N} 张照片" |
| 部分失败 | 橙色警告 | "成功 {X} 张，失败 {Y} 张" |

## 七、自动清理策略

| 任务状态 | 清理时机 | 说明 |
|---|---|---|
| `completed` | 完成后 **8 秒** 自动移除 | 避免已完成任务堆积 |
| `partial` | 不自动清理 | 保留供用户重试 |
| `failed` | 不自动清理 | 保留供用户重试 |

当所有任务被清理后，任务指示条自动隐藏。

## 八、交互时序图

### 正常发送流程

```
用户操作          App 前台 UI                         后台任务
───────          ──────────                         ────────
确认发送    →    照片标记发送中蒙层
                 退出多选模式
                 弹出「开始发送」Banner (3.5s)   →   创建 Task, 开始逐张发送
                 显示任务指示条（蓝色进度）       ←   每张完成后更新 progress
                     ...                              ...
                                                      全部处理完
                 弹出「完成」Banner (4s)          ←   状态 → completed/partial/failed
                 指示条更新状态
                                                      8s 后自动清理 completed 任务
                 指示条隐藏（若无剩余任务）
```

### 失败重试流程

```
用户操作              App 前台 UI                      后台任务
───────              ──────────                      ────────
点击指示条      →    打开任务面板
点击「重试」    →    面板内新增 sending 行          →  新建 Task（仅失败照片）
                     照片重新标记发送中蒙层              开始逐张发送
                     指示条恢复蓝色进度              ←  每张完成后更新 progress
                         ...                            ...
                     弹出「完成」Banner              ←  发送完毕
```

## 附：颜色与时间参数速查

### 颜色

| 用途 | 色值 |
|---|---|
| 发送中 / 主色 | `#007AFF` |
| 成功 | `#34C759` |
| 部分失败 / 警告 | `#FF9500` |
| 全部失败 / 错误 | `#FF3B30` |

### 时间参数

| 参数 | 值 | 说明 |
|---|---|---|
| 开始 Banner 展示时长 | 3.5 秒 | 自动消失 |
| 完成 Banner 展示时长 | 4 秒 | 自动消失 |
| 已完成任务自动清理 | 8 秒 | 从任务列表中移除 |
| 后台通知栏刷新频率 | 500 毫秒 | 实时更新进度 |
| Push 通知弹出延迟 | 600 毫秒 | 进度栏隐藏后延迟弹出 |
| 任务指示条最小点击高度 | ≥ 44pt | 符合 HIG 规范 |
