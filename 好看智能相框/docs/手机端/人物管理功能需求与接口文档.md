# 人物管理功能需求与接口文档

本文档详细描述了人物相册管理相关的功能需求、交互流程及后端接口规范。

主要包含以下功能：
1.  **合并人物**：解决同一个人被识别为多个人物的问题。
2.  **隐藏人物**：隐藏不想在首页展示的人物。
3.  **重置人物建议**：恢复初始状态，找回隐藏人物或撤销合并。

---

## 1. 功能总览

| 功能模块 | 功能名称 | 入口位置 | 功能说明 |
| :--- | :--- | :--- | :--- |
| **人像合辑** | **合并人物** | 人像详情页 -> 右上角菜单 -> 合并 | 将当前人物合并到另一个人物，合并后当前人物消失，照片归入目标人物。 |
| **人像合辑** | **隐藏人物** | 人像详情页 -> 右上角菜单 -> 隐藏 | 隐藏后该人像不再出现在“人物”列表首页，但照片不删除。 |
| **系统设置** | **重置人物建议** | 设置页 -> 重置人物建议 | 恢复所有已隐藏的人物，取消已合并的状态（如果支持），并重新触发识别建议。 |

## 2. 合并人物 (Merge Face)

### 2.1 功能说明
当算法将同一个人识别为两个不同的人物（例如“张三”和“未命名1”）时，用户可以通过此功能将它们合并。

*   **操作对象**：源人物（Source，当前查看的人物）、目标人物（Target，要合并到的人物）。
*   **结果**：
    *   源人物（Source）将从人物列表中消失。
    *   源人物下的所有照片将移动到目标人物（Target）名下。
    *   目标人物的照片数量 = 原照片数 + 源人物照片数。

### 2.2 交互流程
1.  用户进入某个人物（Source）的详情页。
2.  点击右上角“更多”菜单，选择**“合并”**。
3.  系统弹出**“选择人物”**对话框，列出其他所有人物（Target）供选择。
4.  用户选择一个目标人物，点击确认。
5.  调用接口进行合并。
6.  合并成功后，返回上一级（人物列表页）或跳转到目标人物详情页。

### 2.3 接口定义

*   **接口地址**: `/v1/photoFrame/mergeFace`
*   **请求方式**: `POST`
*   **Content-Type**: `application/json`

#### 请求参数

| 参数名 | 类型 | 必填 | 说明 |
| :--- | :--- | :--- | :--- |
| `userId` | String | 是 | 当前登录用户 ID |
| `token` | String | 是 | 用户访问令牌 |
| `masterFaceId` | String | 是 | **保留的主人物 ID** (Target Face ID)<br>即用户选择要合并到的那个“目标人物” |
| `mergeFaceId` | String | 是 | **被合并的人物 ID** (Source Face ID)<br>即当前操作的这个人物，合并后该 ID 对应的人物将消失 |
| `newFaceName` | String | 否 | 合并后的人物新名称（可选） |

#### 请求示例

```json
{
    "userId": "10086",
    "token": "eyJhbGciOiJIUzI1NiIsInR5cCI...",
    "masterFaceId": "face_target_12345",
    "mergeFaceId": "face_source_67890"
}
```

#### 响应示例

```json
{
    "status": 0,
    "err": "success"
}
```

## 3. 隐藏人物 (Hide Face)

### 3.1 功能说明
对于用户不想在人物相册首页看到的人物（例如路人、识别错误的非人物体等），可以进行隐藏。

*   **效果**：隐藏后，该人物卡片不再显示在“人物”列表中。
*   **数据影响**：照片本身**不会**被删除，依然可以在“所有照片”或“相册”中找到。
*   **恢复**：单个隐藏人物无法直接在列表恢复，需通过全局“重置人物建议”功能统一恢复。

### 3.2 交互流程
1.  用户进入某个人物详情页。
2.  点击右上角“更多”菜单，选择**“隐藏”**。
3.  系统弹出确认对话框：
    > **隐藏确认**
    >
    > 确定要隐藏此人物吗？隐藏后该人物将不再显示在首页，但照片不会被删除。可通过“设置-重置人物建议”恢复。
4.  用户点击“确定”。
5.  调用接口进行隐藏。
6.  成功后返回人物列表页，该人物卡片消失。

### 3.3 接口定义

*   **接口地址**: `/v1/photoFrame/hideFace`
*   **请求方式**: `POST`
*   **Content-Type**: `application/json`

#### 请求参数

| 参数名 | 类型 | 必填 | 说明 |
| :--- | :--- | :--- | :--- |
| `userId` | String | 是 | 当前登录用户 ID |
| `token` | String | 是 | 用户访问令牌 |
| `faceId` | String | 是 | **需要隐藏的人物 ID** |

#### 请求示例

```json
{
    "userId": "10086",
    "token": "eyJhbGciOiJIUzI1NiIsInR5cCI...",
    "faceId": "face_12345"
}
```

#### 响应示例

```json
{
    "status": 0,
    "err": "success"
}
```

## 4. 重置人物建议 (Reset Face Suggestions)

### 4.1 功能说明
这是一个全局重置功能，用于撤销用户之前对人物相册所做的手动干预操作。

*   **入口**：App 设置页面。
*   **效果**：
    1.  **恢复隐藏**：所有被标记为“隐藏”的人物将重新显示。
    2.  **重置建议**：清除用户对系统智能建议（如“这是同一个人吗？”）的忽略记录。
    3.  **(视具体实现)** 可能包含取消之前的合并操作，恢复原始识别状态。
*   **场景**：用户误隐藏了重要人物，或者想要重新整理人物相册。

### 4.2 交互流程
1.  用户进入“设置”页面。
2.  点击**“重置人物建议”**选项。
3.  系统弹出确认对话框：
    > **重置人物建议**
    >
    > 确定要重置吗？重置后将恢复所有已隐藏的人物，并重新进行智能分类建议。
4.  用户点击“确定”。
5.  调用接口执行重置。
6.  提示“重置成功”。

### 4.3 接口定义

*   **接口地址**: `/v1/photoFrame/resetFace`
*   **请求方式**: `POST`
*   **Content-Type**: `application/json`

#### 请求参数

| 参数名 | 类型 | 必填 | 说明 |
| :--- | :--- | :--- | :--- |
| `userId` | String | 是 | 当前登录用户 ID |
| `token` | String | 是 | 用户访问令牌 |

#### 请求示例

```json
{
    "userId": "10086",
    "token": "eyJhbGciOiJIUzI1NiIsInR5cCI..."
}
```

#### 响应示例

```json
{
    "status": 0,
    "err": "success"
}
```

## 6. 注意事项

1.  **数据安全**：
    *   **隐藏 ≠ 删除**：务必向用户明确，隐藏操作不会删除照片文件。
    *   **合并不可逆**：通常合并操作是不可逆的（除非通过完全重置），操作前必须给用户明确的后果提示。

2.  **网络状态**：
    *   这些操作都强依赖服务端接口，需处理网络请求失败的情况（如 Loading 状态、Toast 提示失败原因）。

3.  **本地缓存更新**：
    *   操作成功后，客户端应及时刷新本地的人脸列表数据，确保 UI 与服务器状态同步。
