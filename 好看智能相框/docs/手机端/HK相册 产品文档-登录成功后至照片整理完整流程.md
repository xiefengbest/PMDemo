# HK相册 产品文档
## 登录成功后至照片整理完整流程

**文档版本：** v1.0.3  
**更新日期：** 2025年12月  
**文档类型：** 产品需求文档（PRD）

---

## 一、功能概述

本文档描述用户登录成功后，首次进入应用到完成照片整理的完整用户旅程。核心目标是引导用户完成相框绑定、授权照片权限，并自动整理用户的本地照片。

---

## 三、页面功能详述

### 3.1 应用主页

| 项目 | 说明 |
|------|------|
| **页面位置** | 登录成功后的落地页 |
| **页面结构** | 底部Tab导航，共3个Tab |
| **默认Tab** | 首页 |

**底部导航Tab：**

| Tab | 名称 | 功能说明 |
|-----|------|----------|
| 1 | 首页 | 照片预览/整理入口 |
| 2 | 商城 | 商品展示和购买 |
| 3 | 我的 | 个人中心和设置 |

---

### 3.2 首页 - 新用户视角（照片预览页）

**触发条件：** 用户从未发送过照片

#### 3.2.1 页面状态：未绑定相框

**页面元素：**
- 顶部标题栏
- 绑定引导图片/动画
- 引导文案
- 【扫码绑定】按钮

**用户操作：**

| 操作 | 结果 |
|------|------|
| 点击【扫码绑定】 | 打开扫码页面，扫描相框二维码完成绑定 |

**说明：** 根据最新需求，已发送过照片的用户将不再显示"开始整理"横幅，直接进入整理页。

---

#### 3.2.2 页面状态：已绑定相框 + 无照片权限

**页面元素：**
- 顶部标题栏（显示"全部照片"）
- 权限说明占位图
- 【授权访问】按钮

**权限申请流程：**

---

#### 3.2.3 页面状态：已绑定相框 + 有照片权限

**页面元素：**
- 顶部标题栏（可点击切换文件夹）
- 照片宫格列表（4列）
- 右上角【发送】按钮

**核心功能：**

| 功能 | 说明 |
|------|------|
| 照片浏览 | 宫格展示本地全部照片 |
| 文件夹切换 | 点击标题可切换不同相册文件夹 |
| 多选模式 | 默认开启，最多选择10张 |
| 发送照片 | 选中照片后发送到已绑定的相框 |

**发送照片流程：**

```
选择照片（1-10张）
        │
        ▼
  点击【发送】按钮
        │
        ▼
┌───────────────────────┐
│    选择相框弹窗        │
│                       │
│  ☑ 客厅相框           │
│  ☐ 卧室相框           │
│                       │
│  [取消]    [确认]     │
└───────────────────────┘
        │
        ▼
   上传进度弹窗
        │
        ▼
   发送成功提示
        │
        ▼
   切换到照片整理页
```

**重要说明：** 发送照片成功后，系统会自动保存"已发送过照片"标志，下次进入首页将直接显示整理页，不再显示预览页。

---

### 3.3 首页 - 老用户视角（照片整理页）

**触发条件：** 用户已发送过照片

#### 3.3.1 页面布局

**页面元素：**
- 顶部标题栏
  - 左侧：标题"我的照片"
  - 右侧：【添加】【扫码】按钮
- 月份照片列表（按时间倒序排列）
- 底部加载状态

#### 3.3.2 月份卡片展示

每个月份卡片包含：

| 元素 | 说明 |
|------|------|
| 封面图 | 该月份第一张有效照片 |
| 月份标题 | 如"2024年12月" |
| 照片数量 | 该月份有效照片数量 |
| 整理进度 | 识别进行中显示进度条 |

**有效照片定义：**
- 非截图
- 非文档扫描件
- 非纯文字图片
- 未删除
- 未发布

#### 3.3.3 照片整理流程

```
进入照片整理页
        │
        ▼
  检查照片权限  ──────→ 无权限 → 请求权限
        │
        │ 有权限
        ▼
   开始扫描本地照片
        │
        ▼
┌─────────────────────────────────────┐
│          分页扫描策略                │
│                                     │
│  1. 按时间倒序扫描（最新→最旧）       │
│  2. 每批扫描200张                   │
│  3. 按月份自动分组                   │
│  4. 优先展示最新月份                 │
│  5. AI识别照片类型                   │
│  6. 过滤无效照片                     │
└─────────────────────────────────────┘
        │
        ▼
   实时更新月份列表
        │
        ▼
   整理完成
```

**整理过程中UI表现：**
- 月份卡片逐个出现（最新的先出现）
- 每个月份显示整理进度
- 列表底部显示"正在整理本地照片..."加载状态
- 整理完成后加载状态消失

**状态提示：**
- **整理中**：显示"照片整理中"状态条，按钮显示"停止整理"
- **整理完成**：显示"整理完成"状态条，按钮显示"重新整理"

## 四、核心业务规则

### 4.1 用户类型判断

| 用户类型 | 判断条件 | 首页展示 |
|----------|----------|----------|
| 新用户 | 从未发送过照片 | 照片预览页 |
| 老用户 | 已发送过照片 | 照片整理页 |

**判断逻辑：**
- 通过 SharedPreferences 中的 `has_ever_started_analysis` 标志判断
- **核心判断依据：是否发送过照片**
- 发送照片成功时会写入标志
- 点击"开始整理"按钮时也会写入标志（但主要判断依据是发送照片）

### 4.2 相框绑定状态

| 状态 | 展示内容 | 可用功能 |
|------|----------|----------|
| 未绑定 | 绑定引导页 | 扫码绑定 |
| 已绑定 | 照片选择页（新用户）或整理页（老用户） | 完整功能 |

### 4.3 照片选择规则

| 规则项 | 限制 |
|--------|------|
| 最大选择数量 | 10张 |
| 选择方式 | 点击照片勾选/取消 |
| 达到上限提示 | Toast提示"最多选择10张" |

### 4.4 照片过滤规则

整理时自动过滤以下类型的照片：

| 过滤类型 | 说明 |
|----------|------|
| 截图 | 屏幕截图 |
| 文档 | 文档扫描件 |
| 纯文字 | 只有文字的图片 |
| 已删除 | 用户标记删除的照片 |
| 已发布 | 已发送到相框的照片 |

## 五、异常处理

### 5.1 网络异常

| 场景 | 处理方式 |
|------|----------|
| 获取相框列表失败 | 默认显示绑定引导页 |
| 发送照片失败 | Toast提示"发送失败，请检查网络" |
| 登录接口失败 | Toast提示错误信息 |

### 5.2 权限异常

| 场景 | 处理方式 |
|------|----------|
| 用户拒绝权限 | 显示权限占位页，提供重新申请入口 |
| 用户永久拒绝 | 弹窗引导用户前往系统设置开启 |

### 5.3 账号异常

| 场景 | 处理方式 |
|------|----------|
| 账号注销中 | 弹窗询问是否恢复账号 |
| 账号被封禁 | 弹窗显示封禁原因和解封时间 |

## 六、数据埋点建议

| 事件名称 | 触发时机 | 关键参数 |
|----------|----------|----------|
| login_success | 登录成功 | user_id |
| frame_bindPage_show | 绑定引导页展示 | - |
| frame_bindBtn_click | 点击扫码绑定 | - |
| frame_bind_success | 绑定成功 | device_id |
| permission_request | 请求照片权限 | - |
| permission_granted | 权限授权成功 | - |
| permission_denied | 权限授权失败 | is_permanent |
| analysis_start | 开始照片整理 | - |
| analysis_complete | 整理完成 | photo_count, month_count |
| photo_send | 发送照片到相框 | photo_count, album_ids |

**说明：** 埋点事件名称保持技术术语（analysis），但用户可见文案统一使用"整理"。
